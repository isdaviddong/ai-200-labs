將容器化後端 API 部署到 Azure Container Apps

在本練習中，您會將容器化後端 API 部署到 Azure Container Apps。您會使用受控識別，安全地從 Azure Container Registry 提取映像，並將祕密設定為環境變數。

本練習中將完成下列工作：

  • 下載專案起始檔並部署 Azure 服務
  • 使用受控識別驗證部署容器應用程式
  • 設定祕密並從環境變數參考它們
  • 透過呼叫 API 端點與檢視記錄來驗證部署

本練習約需 30 分鐘完成。

重要： Azure Container Registry task runs 目前不支援 Azure 免費額度。本練習需要 Pay-As-You-Go 或其他付費方案。

開始之前

若要完成本練習，您需要：

  • 具備部署所需 Azure 服務權限的 Azure 訂閱。如果您還沒有，可以 註冊一個。
  • Visual Studio Code ，可在下列其中一個 支援的平台上執行。
  • 最新版的 Azure CLI。
  • 選用： Python 3.12 或更新版本。

下載專案起始檔並部署 Azure 服務

在本節中，您會下載專案起始檔，並使用指令碼將所需服務部署到 Azure 訂閱。Azure Container Registry 與 Container Apps 環境的部署需要幾分鐘才能完成。

  1. 開啟瀏覽器並輸入下列 URL 以下載起始檔。檔案會儲存在您的預設下載位置。

    https://github.com/MicrosoftLearning/mslearn-azure-ai/raw/main/downloads/python/aca-deploy-python.zip

  2. 將檔案複製或移動到您希望進行專案作業的系統位置，然後將其解壓縮到資料夾中。

  3. 啟動 Visual Studio Code (VS Code)，並選取 File > Open Folder... ，然後選擇包含專案檔案的資料夾。

  4. 此專案同時包含 Bash（azdeploy.sh）與 PowerShell（azdeploy.ps1）。開啟符合您環境的檔案，將指令碼頂端的兩個值修改為符合您的需求，然後儲存變更。 注意： 不要變更指令碼中的其他任何內容。

    "<your-resource-group-name>" # Resource Group name
"<your-azure-region>" # Azure region for the resources

  5. 在功能表列中選取 Terminal > New Terminal ，在 VS Code 中開啟終端機視窗。

  6. 執行下列命令登入您的 Azure 帳戶。依照提示選取本練習要使用的 Azure 帳戶與訂閱。

    az login

  7. 執行下列命令以確認您已安裝 Azure CLI 的 containerapp 延伸模組。

    az extension add --name containerapp

  8. 執行下列命令，以確保您的訂閱具有本練習所需的資源提供者。

    az provider register --namespace Microsoft.App
az provider register --namespace Microsoft.OperationalInsights
az provider register --namespace Microsoft.ContainerRegistry

在 Azure 中建立資源

在本節中，您會執行部署指令碼，將所需服務部署到您的 Azure 訂閱。

  1. 請確認您位於專案根目錄，並在終端機中執行對應命令來啟動部署指令碼。部署指令碼會部署 ACR，並建立本練習所需環境變數的檔案。

    Bash

    bash azdeploy.sh

    PowerShell

    ./azdeploy.ps1

    注意： 如果 PowerShell 因指令碼未經數位簽署而封鎖它，請在同一個終端機工作階段中執行下列命令，然後再次執行部署指令碼。此命令只會變更目前 PowerShell 處理序的執行原則。

    Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass

  2. 當指令碼執行時，輸入 1 以啟動 Create Azure Container Registry and build container image 選項。此選項會建立 ACR 服務，並使用 ACR Tasks 建置與推送映像到登錄。

  3. 前一個作業完成後，輸入 2 以啟動 Create Container Apps environment 選項。部署容器之前必須先建立環境。

    注意： 建立 Container Apps 環境後，系統會產生一個包含環境變數的檔案。您會在整個練習中使用這些變數。

  4. 前一個作業完成後，輸入 4 以結束部署指令碼。

  5. 執行對應命令，從前一步建立的檔案中將環境變數載入目前的終端機工作階段。

    Bash

    source .env

    PowerShell

    . .\.env.ps1

    注意： 請保持終端機開啟。如果您將它關閉並建立新的終端機，可能需要再次執行命令來建立環境變數。

部署容器應用程式並設定祕密

在本節中，您會將 API 以啟用外部輸入的容器應用程式形式部署。由於映像位於私人登錄中，您必須在建立時設定登錄驗證，讓第一個修訂可以提取映像。接著，您會設定祕密並從環境變數參考它。此模式反映了 AI 應用程式儲存提供者 API 金鑰的方式。

  1. 使用系統指派的受控識別建立容器應用程式，並在建立時設定登錄驗證。 --registry-identity 旗標會指示 Container Apps 使用應用程式的受控識別，從指定的登錄提取映像。當您搭配 Azure Container Registry 使用此旗標時，CLI 會自動指派 AcrPull 角色。

    Bash

    az containerapp create \
    --name $CONTAINER_APP_NAME \
    --resource-group $RESOURCE_GROUP \
    --environment $ACA_ENVIRONMENT \
    --image "$ACR_SERVER/$CONTAINER_IMAGE" \
    --ingress external \
    --target-port $TARGET_PORT \
    --env-vars MODEL_NAME=$MODEL_NAME \
    --registry-server "$ACR_SERVER" \
    --registry-identity system

    PowerShell

    az containerapp create `
    --name $env:CONTAINER_APP_NAME `
    --resource-group $env:RESOURCE_GROUP `
    --environment $env:ACA_ENVIRONMENT `
    --image "$env:ACR_SERVER/$env:CONTAINER_IMAGE" `
    --ingress external `
    --target-port $env:TARGET_PORT `
    --env-vars MODEL_NAME=$env:MODEL_NAME `
    --registry-server "$env:ACR_SERVER" `
    --registry-identity system

  2. 建立祕密並從環境變數參考它。

    Bash

    az containerapp secret set -n $CONTAINER_APP_NAME -g $RESOURCE_GROUP \
    --secrets embeddings-api-key=$EMBEDDINGS_API_KEY

    PowerShell

    az containerapp secret set -n $env:CONTAINER_APP_NAME -g $env:RESOURCE_GROUP `
    --secrets embeddings-api-key=$env:EMBEDDINGS_API_KEY

  3. 從環境變數參考該祕密。此命令會建立新的修訂，並重新啟動應用程式，讓祕密變更生效。

    Bash

    az containerapp update -n $CONTAINER_APP_NAME -g $RESOURCE_GROUP \
    --set-env-vars EMBEDDINGS_API_KEY=secretref:embeddings-api-key

    PowerShell

    az containerapp update -n $env:CONTAINER_APP_NAME -g $env:RESOURCE_GROUP `
    --set-env-vars EMBEDDINGS_API_KEY=secretref:embeddings-api-key

  4. 執行下列命令以列出修訂，確認已建立新的修訂。

    az containerapp revision list -n $CONTAINER_APP_NAME -g $RESOURCE_GROUP -o table

    修訂名稱會以類似 --0000002的尾碼結尾，表示這是第二個修訂。每當您變更環境變數或祕密時，Container Apps 都會建立新的修訂，並使用更新後的設定重新啟動應用程式。舊的非作用中修訂可能會隨時間被清理。

驗證部署

您應驗證應用程式是否已啟動，且輸入設定是否正常運作。您也會使用記錄來確認應用程式的行為符合預期。

  1. 執行下列命令以取得應用程式的 FQDN，並將結果儲存到變數中。

    Bash

    FQDN=$(az containerapp show -n $CONTAINER_APP_NAME -g $RESOURCE_GROUP \
    --query properties.configuration.ingress.fqdn -o tsv)

echo "$FQDN"

    PowerShell

    $FQDN = az containerapp show -n $env:CONTAINER_APP_NAME -g $env:RESOURCE_GROUP `
    --query properties.configuration.ingress.fqdn -o tsv

Write-Output $FQDN

  2. 執行下列命令以呼叫健康狀態端點。此命令應回傳 {"status": "healthy"}.

    Bash

    curl -s "https://$FQDN/health"

    PowerShell

    Invoke-RestMethod -Uri "https://$FQDN/health"

  3. 執行下列命令，透過呼叫根端點來驗證祕密是否已設定。該端點會回傳 JSON，其中包含應用程式資訊，例如已設定的模型名稱，以及 API 金鑰祕密是否已設定。

    Bash

    curl -s "https://$FQDN/"

    PowerShell

    Invoke-RestMethod -Uri "https://$FQDN/"

  4. 執行下列命令以測試文件處理端點。此命令會將 document.txt 檔案傳送到該端點。作業會回傳包含模擬資料分析資訊的 JSON。

    Bash

    curl -s -X POST "https://$FQDN/process" \
    -H "Content-Type: text/plain" \
    -d @document.txt

    PowerShell

    Invoke-RestMethod -Uri "https://$FQDN/process" `
    -Method Post `
    -ContentType "text/plain" `
    -Body (Get-Content -Raw document.txt)

  5. 執行下列命令以檢閱啟動與執行階段訊號的記錄。此命令只會顯示最近的主控台輸出。若要進行歷史記錄檢視與進階疑難排解，記錄會保留在與您的 Container Apps 環境相關聯的 Log Analytics 工作區中。

    az containerapp logs show -n $CONTAINER_APP_NAME -g $RESOURCE_GROUP

    請尋找 gunicorn 啟動訊息，內容會顯示工作程序已啟動並正在接聽連接埠 8000。您也應該能看到來自 curl 命令的 HTTP 要求記錄（GET /health、POST /process 等）。

清除資源

完成本練習後，您應刪除所建立的雲端資源，以避免不必要的資源使用。

  1. 在 VS Code 終端機中執行下列命令以刪除資源群組及其中所有資源。將 <rg-name> 替換為您在本練習稍早選擇的名稱。此命令會在 Azure 中啟動背景工作來刪除資源群組。

    az group delete --name <rg-name> --no-wait --yes

注意： 刪除資源群組會刪除其中包含的所有資源。如果您在本練習中選擇的是現有資源群組，則本練習範圍外的現有資源也會一併被刪除。

疑難排解

若您在完成本練習時遇到問題，請嘗試下列疑難排解步驟：

使用指令碼檢查部署狀態

  • 執行部署指令碼並選取選項 3 以檢查您的 ACR 與 Container Apps 環境狀態。這可驗證基礎基礎結構已部署，且容器映像存在。

驗證 Azure 驗證與環境變數

  • 執行 az account show 以確認您已登入正確的 Azure 訂閱。
  • 執行下列命令以確認環境變數已設定： echo $ACR_NAME （Bash）或 $env:ACR_NAME （PowerShell）。
  • 如果變數為空，請重新執行 source .env （Bash）或 . ..env.ps1 （PowerShell）。

驗證 ACR 部署

  • 前往 Azure portal 並找到您的資源群組。
  • 確認 Azure Container Registry 存在，且 Provisioning State 為 Succeeded.
  • 執行 az acr list --output table 以確認您的登錄可供存取。

排解建置失敗

  • 部署指令碼會抑制冗長的 az acr build 輸出。若要排解失敗，請檢查最近一次 ACR Task 執行的狀態與記錄。
  • 確認您是從專案根目錄執行部署指令碼（ api 資料夾所在的位置）。
  • 列出最近的 ACR Task 執行：
    • Bash： az acr task list-runs --registry $ACR_NAME --output table
    • PowerShell： az acr task list-runs --registry $env:ACR_NAME --output table
  • 檢視特定執行的記錄（將 <run-id> 替換為上一個命令中的值）：
    • Bash： az acr task logs --registry $ACR_NAME --run-id <run-id>
    • PowerShell： az acr task logs --registry $env:ACR_NAME --run-id <run-id>

排解容器提取失敗（ImagePullBackOff / unauthorized / 403）

  • 確認容器應用程式已啟用系統指派的受控識別：
    • Bash： az containerapp identity show -n $CONTAINER_APP_NAME -g $RESOURCE_GROUP
    • PowerShell： az containerapp identity show -n $env:CONTAINER_APP_NAME -g $env:RESOURCE_GROUP
  • 確認容器應用程式具有 AcrPull 且角色指派已套用到登錄範圍。角色指派建立後可能需要一到兩分鐘才會完成傳播。

排解容器啟動與應用程式錯誤

  • 串流檢視容器記錄以診斷啟動問題：
    • Bash： az containerapp logs show -n $CONTAINER_APP_NAME -g $RESOURCE_GROUP --follow
    • PowerShell： az containerapp logs show -n $env:CONTAINER_APP_NAME -g $env:RESOURCE_GROUP --follow
  • 如果應用程式在部署後不久回傳 502/503，請等一分鐘後再試。首次啟動可能需要較長時間，因為 Container Apps 正在提取並啟動容器。
  • 檢查修訂狀態以確認是否有佈建錯誤：
    • Bash： az containerapp revision list -n $CONTAINER_APP_NAME -g $RESOURCE_GROUP -o table
    • PowerShell： az containerapp revision list -n $env:CONTAINER_APP_NAME -g $env:RESOURCE_GROUP -o table

排解祕密設定

  • 驗證祕密已建立：
    • Bash： az containerapp secret list -n $CONTAINER_APP_NAME -g $RESOURCE_GROUP -o table
    • PowerShell： az containerapp secret list -n $env:CONTAINER_APP_NAME -g $env:RESOURCE_GROUP -o table
  • 呼叫根端點（/），以確認環境變數是否正確參考祕密，並查看 API 金鑰是否已設定。
