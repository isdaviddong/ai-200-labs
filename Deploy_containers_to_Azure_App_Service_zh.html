將容器部署到 Azure App Service

在本練習中，您會將 Linux 容器映像從 Azure Container Registry (ACR) 部署到 Azure App Service。您會將 Web App 設定為使用系統指派的受控識別與 AcrPull 角色，讓 App Service 能從您的私人登錄提取映像，而不必在應用程式設定中儲存登錄認證。

本練習中將完成下列工作：

  • 下載專案起始檔
  • 部署 Azure Container Registry 並使用 ACR Tasks 建置容器映像
  • 部署適用於 Linux 容器的 App Service 方案
  • 建立並設定 Web App for Containers，使其透過受控識別從 ACR 提取映像
  • 設定執行階段設定並啟用容器記錄
  • 驗證部署並測試文件處理端點

本練習約需 30 分鐘完成。

重要： Azure Container Registry task runs 目前不支援 Azure 免費額度。本練習需要 Pay-As-You-Go 或其他付費方案。

開始之前

若要完成本練習，您需要：

  • 具備部署所需 Azure 服務權限的 Azure 訂閱。如果您還沒有，可以 註冊一個。
  • Visual Studio Code ，可在下列其中一個 支援的平台上執行。
  • 最新版的 Azure CLI。
  • 選用： Python 3.12 或更新版本。

下載專案起始檔並部署 Azure 服務

在本節中，您會下載專案起始檔，並使用指令碼將所需服務部署到 Azure 訂閱。Azure Container Registry 與 App Service 方案的部署需要幾分鐘才能完成。

  1. 開啟瀏覽器並輸入下列 URL 以下載起始檔。檔案會儲存在您的預設下載位置。

    https://github.com/MicrosoftLearning/mslearn-azure-ai/raw/main/downloads/python/app-svc-container-python.zip

  2. 將檔案複製或移動到您希望進行專案作業的系統位置，然後將其解壓縮到資料夾中。

  3. 啟動 Visual Studio Code (VS Code)，並選取 File > Open Folder... ，然後選擇包含專案檔案的資料夾。

  4. 此專案同時包含 Bash（azdeploy.sh）與 PowerShell（azdeploy.ps1）。開啟符合您環境的檔案，將指令碼頂端的兩個值修改為符合您的需求，然後儲存變更。 注意： 不要變更指令碼中的其他任何內容。

    "<your-resource-group-name>" # Resource Group name
"<your-azure-region>" # Azure region for the resources

  5. 在功能表列中選取 Terminal > New Terminal ，在 VS Code 中開啟終端機視窗。

  6. 執行下列命令登入您的 Azure 帳戶。依照提示選取本練習要使用的 Azure 帳戶與訂閱。

    az login

  7. 執行下列命令，以確保您的訂閱具備 Azure Container Registry (ACR) 與 Azure App Service 所需的資源提供者。

    az provider register --namespace Microsoft.ContainerRegistry
az provider register --namespace Microsoft.Web

在 Azure 中建立資源

在本節中，您會執行部署指令碼，將所需服務部署到您的 Azure 訂閱。

  1. 請確認您位於專案根目錄，並在終端機中執行對應命令來啟動部署指令碼。部署指令碼會部署 ACR，並建立本練習所需環境變數的檔案。

    Bash

    bash azdeploy.sh

    PowerShell

    ./azdeploy.ps1

    注意： 如果 PowerShell 因指令碼未經數位簽署而封鎖它，請在同一個終端機工作階段中執行下列命令，然後再次執行部署指令碼。此命令只會變更目前 PowerShell 處理序的執行原則。

    Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass

  2. 當指令碼執行時，輸入 1 以啟動 1. Create Azure Container Registry and build container image 選項。此選項會建立 ACR 服務，並使用 ACR Tasks 建置與推送映像到登錄。

  3. 前一個作業完成後，輸入 2 以啟動 Create App Service Plan 選項。此選項會建立 Web App 所需的 App Service 方案。

    注意： 建立 App Service 方案後，系統會產生一個包含環境變數的檔案。您會在整個練習中使用這些變數。

  4. 前一個作業完成後，輸入 4 以結束部署指令碼。

  5. 執行對應命令，從前一步建立的檔案中將環境變數載入目前的終端機工作階段。

    Bash

    source .env

    PowerShell

    . .\.env.ps1

    注意： 請保持終端機開啟。如果您將它關閉並建立新的終端機，可能需要再次執行命令來建立環境變數。

建立 Web App

在本節中，您會使用 CLI 命令建立 Web App。接著，您會將 Web App 設定為使用系統指派的受控識別，讓應用程式可以存取 ACR 中的映像。

  1. 執行下列命令以建立已設定為從您的容器登錄提取映像的 Web App for Containers。

    Bash

    az webapp create \
    --resource-group $RESOURCE_GROUP \
    --plan $APP_PLAN \
    --name $APP_NAME \
    --container-image-name $ACR_NAME.azurecr.io/docprocessor:v1

    PowerShell

    az webapp create `
    --resource-group $env:RESOURCE_GROUP `
    --plan $env:APP_PLAN `
    --name $env:APP_NAME `
    --container-image-name "$($env:ACR_NAME).azurecr.io/docprocessor:v1"

    根據預設，您的 Azure Container Registry 是私有的。App Service 在提取映像之前，需要有方法向 ACR 進行驗證。

    您會使用系統指派的受控識別（建議做法）來設定這項驗證，而不是在應用程式設定中儲存登錄認證。

  2. 執行下列命令以在 Web App 上啟用系統指派的受控識別。

    Bash

    az webapp identity assign \
    --resource-group $RESOURCE_GROUP \
    --name $APP_NAME

    PowerShell

    az webapp identity assign `
    --resource-group $env:RESOURCE_GROUP `
    --name $env:APP_NAME

將 AcrPull 角色指派給 Web App

在本節中，您會授與 Web App 從私人登錄提取映像的權限。受控識別是由 Microsoft Entra 支援，且由 Azure 為您建立與管理的識別。當您在 Web App 上啟用系統指派的識別後，App Service 就能以該識別要求權杖。

若要讓 Web App 使用該識別提取映像，您需將內建的 AcrPull 角色指派到您的登錄範圍。這遵循最低權限原則：Web App 可以下載映像，但不能推送或管理登錄。

  1. 執行下列命令以取得 Web App 的主體 ID。

    Bash

    PRINCIPAL_ID=$(az webapp identity show \
    --resource-group $RESOURCE_GROUP \
    --name $APP_NAME \
    --query principalId \
    --output tsv)

    PowerShell

    $PRINCIPAL_ID = az webapp identity show `
    --resource-group $env:RESOURCE_GROUP `
    --name $env:APP_NAME `
    --query principalId `
    --output tsv

  2. 執行下列命令以取得 ACR 的 ID。

    Bash

    ACR_ID=$(az acr show \
    --resource-group $RESOURCE_GROUP \
    --name $ACR_NAME \
    --query id \
    --output tsv)

    PowerShell

    $ACR_ID = az acr show `
    --resource-group $env:RESOURCE_GROUP `
    --name $env:ACR_NAME `
    --query id `
    --output tsv

  3. 執行下列命令以將 AcrPull 角色指派給 Web App。

    Bash

    az role assignment create \
    --assignee $PRINCIPAL_ID \
    --scope $ACR_ID \
    --role AcrPull

    PowerShell

    az role assignment create `
    --assignee $PRINCIPAL_ID `
    --scope $ACR_ID `
    --role AcrPull

    注意： 角色指派可能需要一到兩分鐘才會完成傳播。如果應用程式在此步驟後仍無法立即提取映像，請稍候片刻後再試一次。

  4. 執行下列命令以將 Web App 設定為使用受控識別進行登錄驗證。此設定會指示 App Service 在存取容器登錄時使用 Web App 的受控識別（而不是登錄管理員認證）。

    Bash

    az webapp config set \
    --resource-group $RESOURCE_GROUP \
    --name $APP_NAME \
    --acr-use-identity true \
    --acr-identity [system]

    PowerShell

    az webapp config set `
    --resource-group $env:RESOURCE_GROUP `
    --name $env:APP_NAME `
    --acr-use-identity true `
    --acr-identity [system]

  5. 執行下列命令以更新容器設定，使其透過受控識別使用登錄。此步驟會明確設定 Web App 應使用的映像與登錄 URL。若您之後更新映像標籤，就在這裡讓 Web App 指向新版本。

    Bash

    az webapp config container set \
    --resource-group $RESOURCE_GROUP \
    --name $APP_NAME \
    --container-image-name $ACR_NAME.azurecr.io/docprocessor:v1 \
    --container-registry-url https://$ACR_NAME.azurecr.io

    PowerShell

    az webapp config container set `
    --resource-group $env:RESOURCE_GROUP `
    --name $env:APP_NAME `
    --container-image-name "$($env:ACR_NAME).azurecr.io/docprocessor:v1" `
    --container-registry-url "https://$($env:ACR_NAME).azurecr.io"

設定執行階段設定並啟用容器記錄

在本節中，您會設定執行階段設定並啟用記錄，讓容器執行得更可靠，並協助排解問題。

  1. 執行下列命令以設定容器連接埠。範例映像會接聽連接埠 80（預設值），因此此步驟是在不改變行為的情況下示範此設定。

    Bash

    az webapp config appsettings set \
    --resource-group $RESOURCE_GROUP \
    --name $APP_NAME \
    --settings WEBSITES_PORT=80

    PowerShell

    az webapp config appsettings set `
    --resource-group $env:RESOURCE_GROUP `
    --name $env:APP_NAME `
    --settings WEBSITES_PORT=80

  2. 執行下列命令以為已處理的文件啟用持續性儲存體。此設定會啟用 App Service 儲存體掛載（例如 Linux 容器中的 /home 路徑）。

    Bash

    az webapp config appsettings set \
    --resource-group $RESOURCE_GROUP \
    --name $APP_NAME \
    --settings WEBSITES_ENABLE_APP_SERVICE_STORAGE=true

    PowerShell

    az webapp config appsettings set `
    --resource-group $env:RESOURCE_GROUP `
    --name $env:APP_NAME `
    --settings WEBSITES_ENABLE_APP_SERVICE_STORAGE=true

  3. 執行下列命令以啟用 always-on。always-on 會讓應用程式維持暖機狀態，從而降低冷啟動延遲。

    Bash

    az webapp config set \
    --resource-group $RESOURCE_GROUP \
    --name $APP_NAME \
    --always-on true

    PowerShell

    az webapp config set `
    --resource-group $env:RESOURCE_GROUP `
    --name $env:APP_NAME `
    --always-on true

  4. 執行下列命令以啟用容器記錄。這會擷取容器的 stdout/stderr，讓您可以從 CLI 檢視記錄。

    Bash

    az webapp log config \
    --resource-group $RESOURCE_GROUP \
    --name $APP_NAME \
    --docker-container-logging filesystem

    PowerShell

    az webapp log config `
    --resource-group $env:RESOURCE_GROUP `
    --name $env:APP_NAME `
    --docker-container-logging filesystem

驗證部署

在本節中，您會驗證 Web App 是否正在執行並回應要求。

  1. 執行下列命令以取得 Web App 主機名稱。

    Bash

    APP_URL=$(az webapp show \
    --resource-group $RESOURCE_GROUP \
    --name $APP_NAME \
    --query defaultHostName \
    --output tsv)

echo "Application URL: https://$APP_URL"

    PowerShell

    $APP_URL = az webapp show `
    --resource-group $env:RESOURCE_GROUP `
    --name $env:APP_NAME `
    --query defaultHostName `
    --output tsv

Write-Host "Application URL: https://$APP_URL"

  2. 在瀏覽器中開啟此 URL，以確認應用程式有回應。請保持瀏覽器開啟，稍後在本練習中還會使用。應用程式應回傳表示其正在執行的回應。第一次要求可能會花較久時間，因為 App Service 需要提取容器映像並啟動應用程式。

測試文件處理

在本節中，您會向 API 傳送要求，以確認應用程式正常運作，且結果已寫入持續性儲存體。

  1. 執行下列命令，將專案中附帶的 document.txt 檔案提交到處理端點。

    Bash

    curl -X POST "https://$APP_URL/process" \
    -H "Content-Type: text/plain" \
    --data-binary @document.txt

    PowerShell

    $body = Get-Content -Raw -Path "document.txt"
Invoke-RestMethod -Method Post -Uri "https://$APP_URL/process" -ContentType "text/plain" -Body $body | ConvertTo-Json -Depth 10

    API 會回傳模擬分析結果，包括擷取出的實體、關鍵片語與情緒分析。請注意，回應也會指出結果是否已儲存到持續性儲存體。

  2. 執行下列命令以列出所有已處理的文件。

    Bash

    curl https://$APP_URL/documents

    PowerShell

    Invoke-RestMethod -Uri "https://$APP_URL/documents" | ConvertTo-Json -Depth 10

    如果持續性儲存體已正確啟用，您應該會在清單中看到剛剛處理的文件。

串流檢視容器記錄

在本節中，您會串流檢視容器記錄，以協助排解啟動與要求處理問題。

  1. 執行下列命令以檢視容器的即時記錄。

    Bash

    az webapp log tail \
    --resource-group $RESOURCE_GROUP \
    --name $APP_NAME

    PowerShell

    az webapp log tail `
    --resource-group $env:RESOURCE_GROUP `
    --name $env:APP_NAME

  2. 透過重新整理瀏覽器向應用程式產生更多要求。您應該會看到記錄項目出現在串流中。按 Ctrl+C 可停止串流。

檢查診斷主控台

在本節中，您會開啟 SCM (Kudu) 網站，以檢查設定檢視與常見的記錄位置。

  1. 執行下列命令以輸出 SCM (Kudu) URL。

    Bash

    echo "Kudu URL: https://$APP_NAME.scm.azurewebsites.net"

    PowerShell

    Write-Host "Kudu URL: https://$($env:APP_NAME).scm.azurewebsites.net"

  2. 在瀏覽器中開啟此 URL。於頁面頂端選單中，依序前往：

    1. Environment 以檢視環境變數並確認您的應用程式設定存在。
    2. Bash 以開啟瀏覽器式命令殼層與檔案總管。
    3. 在檔案總管中，前往 /home/LogFiles/ 以檢視記錄檔。輸入 ls 來列出資料夾中的檔案。

    提示： 您也可以使用 Log stream 在上方選單中於瀏覽器檢視記錄，或使用 SSH 選項連線到應用程式容器。

    SCM 網站與您的應用程式容器分開，因此它不會提供容器檔案系統或執行中處理程序的完整檢視。

檢視應用程式設定

在本節中，您會確認已設定的應用程式設定存在。

  1. 執行下列命令以列出應用程式設定。

    Bash

    az webapp config appsettings list \
    --resource-group $RESOURCE_GROUP \
    --name $APP_NAME \
    --output table

    PowerShell

    az webapp config appsettings list `
    --resource-group $env:RESOURCE_GROUP `
    --name $env:APP_NAME `
    --output table

    確認您的設定會與系統提供的設定一起出現在清單中。

清除資源

完成本練習後，您應刪除所建立的雲端資源，以避免不必要的資源使用。

  1. 在 VS Code 終端機中執行下列命令以刪除資源群組及其中所有資源。將 <rg-name> 替換為您在本練習稍早選擇的名稱。此命令會在 Azure 中啟動背景工作來刪除資源群組。

    az group delete --name <rg-name> --no-wait --yes

注意： 刪除資源群組會刪除其中包含的所有資源。如果您在本練習中選擇的是現有資源群組，則本練習範圍外的現有資源也會一併被刪除。

疑難排解

若您在完成本練習時遇到問題，請嘗試下列疑難排解步驟：

驗證 Azure 驗證與環境變數

  • 執行 az account show 以確認您已登入正確的 Azure 訂閱。
  • 執行下列命令以確認環境變數已設定： echo $ACR_NAME （Bash）或 $env:ACR_NAME （PowerShell）。
  • 如果變數為空，請重新執行 source .env （Bash）或 . ..env.ps1 （PowerShell）。

驗證 ACR 部署

  • 前往 Azure portal 並找到您的資源群組。
  • 確認 Azure Container Registry 存在，且 Provisioning State 為 Succeeded.
  • 執行 az acr list --output table 以確認您的登錄可供存取。

排解建置失敗

  • 部署指令碼會抑制冗長的 az acr build 輸出。若要排解失敗，請檢查最近一次 ACR Task 執行的狀態與記錄。
  • 確認您是從專案根目錄執行部署指令碼（ api 資料夾所在的位置）。
  • 列出最近的 ACR Task 執行：
    • Bash： az acr task list-runs --registry $ACR_NAME --output table
    • PowerShell： az acr task list-runs --registry $env:ACR_NAME --output table
  • 檢視特定執行的記錄（將 替換為上一個命令中的值）：
    • Bash： az acr task logs --registry $ACR_NAME --run-id
    • PowerShell： az acr task logs --registry $env:ACR_NAME --run-id

排解容器提取失敗（ImagePullBackOff / unauthorized / 403）

  • 執行下列命令以確認 Web App 已啟用系統指派的受控識別： az webapp identity show.
  • 確認 Web App 具有 AcrPull 且角色指派已套用到登錄範圍。角色指派建立後可能需要一到兩分鐘才會完成傳播。
  • 重新執行容器設定步驟，以確保映像名稱與登錄 URL 正確。

排解容器啟動與應用程式錯誤

  • 確認已啟用容器記錄，然後串流檢視記錄：
    • Bash： az webapp log tail --resource-group $RESOURCE_GROUP --name $APP_NAME
    • PowerShell： az webapp log tail --resource-group $env:RESOURCE_GROUP --name $env:APP_NAME
  • 如果應用程式在部署後不久回傳 502/503，請等一分鐘後再試。首次啟動可能需要較長時間，因為 App Service 正在提取並啟動容器。

驗證持續性儲存體已啟用

  • 確認 WEBSITES_ENABLE_APP_SERVICE_STORAGE 設定存在且設為 true.
  • 在提交文件後呼叫 /documents 端點，以確認結果已寫入持續性儲存體。
