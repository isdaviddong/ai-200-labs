在 Azure Kubernetes Service 上設定應用程式

在本練習中，您將學習如何使用 ConfigMaps 設定 Kubernetes 部署中的非敏感設定、使用 Secrets 儲存敏感憑證，以及使用 PersistentVolumeClaims 提供持續性儲存。您將把容器化 API 部署到 Azure Kubernetes Service (AKS)，使用各種 Kubernetes 資源進行設定，並透過 Python 用戶端應用程式與其互動。

本練習將執行的工作：

  • 下載專案起始檔案
  • 將資源部署到 Azure（ACR、AKS 叢集）
  • 將容器映像建置並推送到 Azure Container Registry
  • 設定 kubectl 認證以存取 AKS 叢集
  • 將更新後的 YAML 檔案套用至 AKS，以建立 pod 並使用 LoadBalancer 公開 API
  • 執行用戶端應用程式以測試 API 端點
  • 檢視儲存在持續性磁碟區中的 API 記錄
  • 清理 Azure 資源

本練習約需 30 分鐘完成。

IMPORTANT: 本練習中的持續性儲存實作僅供示範。若用於記錄，生產環境應用程式應使用 Azure Monitor 或 Application Insights 等集中式記錄解決方案，而不是將記錄儲存在持續性磁碟區中。若確實需要持續性儲存，請實作記錄輪替原則，以避免儲存空間被填滿，否則可能導致容器失敗與 pod 被逐出。

重要： Azure Container Registry 作業執行 目前暫停支援 Azure 免費額度。本練習需要 Pay-As-You-Go 或其他付費方案。

開始之前

若要完成本練習，您需要：

  • 具有部署所需 Azure 服務權限的 Azure 訂用帳戶。若您尚未擁有，可註冊一個。
  • 安裝在其中一個支援平台上的 Visual Studio Code。
  • 最新版的 Azure CLI。
  • Kubernetes 命令列工具 kubectl。
  • 選用：Python 3.12 或更新版本。

下載專案起始檔案並部署 Azure 服務

在本節中，您將下載主控台應用程式的起始檔案，並使用指令碼將所需服務部署到您的 Azure 訂用帳戶。Azure Managed Redis 部署需要 5 到 10 分鐘才能完成。

  1. 開啟瀏覽器並輸入下列 URL 以下載起始檔案。檔案會儲存到您的預設下載位置。

    https://github.com/MicrosoftLearning/mslearn-azure-ai/raw/main/downloads/python/aks-configure-python.zip

  2. 將該檔案複製或移動到系統中您想要進行此專案工作的資料夾，然後將檔案解壓縮到資料夾中。

  3. 啟動 Visual Studio Code (VS Code)，並在功能表中選取 File > Open Folder...，然後選擇包含專案檔案的資料夾。

  4. 此專案同時包含 Bash（azdeploy.sh）與 PowerShell（azdeploy.ps1）的部署指令碼。請開啟適用於您環境的檔案，並變更指令碼頂端的兩個值以符合您的需求，然後儲存變更。Note: 不要變更指令碼中的任何其他內容。

    "<your-resource-group-name>" # Resource Group name
"<your-azure-region>" # Azure region for the resources

  5. 在功能表列中選取 Terminal > New Terminal，在 VS Code 中開啟終端機視窗。

  6. 執行下列命令以登入您的 Azure 帳戶。依照提示選取您用於本練習的 Azure 帳戶與訂用帳戶。

    az login

  7. 執行下列命令，以確保您的訂用帳戶具有安裝 AKS 和 ACR 所需的資源提供者。Microsoft.Compute、Microsoft.Network 與 Microsoft.Storage 提供者是 AKS 的相依項目，Azure 通常會自動註冊它們，但明確註冊可避免新訂用帳戶偶爾發生的叢集建立失敗。

    az provider register --namespace Microsoft.ContainerService
az provider register --namespace Microsoft.ContainerRegistry
az provider register --namespace Microsoft.Compute
az provider register --namespace Microsoft.Network
az provider register --namespace Microsoft.Storage

  8. 請確認您位於專案根目錄，然後在終端機中執行適當的命令以啟動部署指令碼。

    Bash

    bash azdeploy.sh

    PowerShell

    ./azdeploy.ps1

    Note: 如果 PowerShell 因指令碼未經數位簽署而封鎖執行，請在相同的終端機工作階段中執行下列命令，然後再次執行部署指令碼。此命令只會變更目前 PowerShell 處理序的執行原則。

    Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass

將資源部署到 Azure

當部署指令碼執行時，請依照下列步驟在 Azure 中建立所需資源。

  1. 模型部署完成後，輸入 1 以啟動 Create Azure Container Registry (ACR)。這會建立 API 容器儲存所在的資源，之後 AKS 資源會從中提取容器。

    當作業完成時，會傳回 ACR 端點。請將此資訊複製下來，稍後在本練習中會用到。

  2. ACR 資源建立完成後，輸入 2 以啟動 Build and push API image to ACR。此選項使用 ACR tasks 來建置映像並將其新增至 ACR 存放庫。此作業可能需要 3 到 5 分鐘才能完成。

  3. 映像建置並推送到 ACR 後，輸入 3 以啟動 Create AKS cluster 選項。這會建立已設定受控識別的 AKS 資源，授與服務從 ACR 資源提取映像的權限，並指派寫入持續性儲存所需的 RBAC 角色。此作業可能需要 5 到 10 分鐘才能完成。

  4. AKS 叢集部署完成後，輸入 4 以啟動 Get AKS credentials for kubectl 選項。這會使用 az aks get-credentials 命令來取得認證並設定 kubectl。

  5. 認證設定完成後，輸入 5 以啟動 Check deployment stats 選項。此選項會回報各項資源是否都已成功部署。

    若所有服務都傳回 successful 訊息，請輸入 6 以結束部署指令碼。

接下來，您將完成把 API 部署到 AKS 所需的 YAML 檔案。

完成 YAML 部署檔案並部署到 AKS

在本節中，您將完成位於 k8s 資料夾中的 YAML 檔案，這些檔案用於設定具持續性儲存的 Kubernetes 部署，以及儲存敏感與非敏感設定。

完成 ConfigMap YAML 檔案

ConfigMaps 會以 pod 可使用的鍵值組形式儲存非敏感設定資料。在本節中，您將建立一個 ConfigMap 來儲存學生姓名、API 版本和記錄路徑等應用程式設定。

  1. 開啟 k8s/configmap.yaml 檔案，並將下列程式碼加入檔案中。如果您想要，也可以將 STUDENT_NAME 的值更新為您的名字。

    apiVersion: v1
kind: ConfigMap
metadata:
  name: api-config
  labels:
    app: aks-config-api # Label for the AKS configuration API
data:
  # Store non-sensitive configuration values
  STUDENT_NAME: "YourNameHere"
  API_VERSION: "1.0.0"
  LOG_PATH: "/var/log/api" # Path for API logs

  2. 閱讀程式碼中的註解，然後儲存變更。

完成 Secrets YAML 檔案

Secrets 會以 base64 編碼格式儲存密碼、權杖和金鑰等敏感資訊。在本節中，您將建立一個 Secret 來儲存 API 在執行階段將存取的敏感憑證。

  1. 開啟 k8s/secrets.yaml 檔案，並將下列程式碼加入檔案中。

    apiVersion: v1
kind: Secret
metadata:
  name: api-secrets
  labels:
    app: aks-config-api
type: Opaque
stringData:
  # Store sensitive credentials as base64-encoded values
  secret-endpoint: "SecretEndpointValue"
  secret-access-key: "SecretAccessKey123456"

  2. 花幾分鐘閱讀程式碼中的註解，然後儲存變更。

完成 PVC YAML 檔案

PersistentVolumeClaim (PVC) 會向 Azure 要求 pod 可掛載的儲存資源。在本節中，您將建立一個 PVC，使用 Azure Disk 儲存體讓 API 記錄檔在 pod 重新啟動後仍能保留。

  1. 開啟 k8s/pvc.yaml 檔案，並將下列程式碼加入檔案中。

    apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: api-logs-pvc
  labels:
    app: aks-config-api # Label for the AKS configuration API
spec:
  accessModes:
    - ReadWriteOnce  # Allow single pod to mount volume for read/write
  resources:
    requests:
      storage: 1Gi  # Request minimum Azure Disk size
  storageClassName: managed-csi  # Use Azure Disk CSI driver (default)
  volumeMode: Filesystem  # Default mode

  2. 花幾分鐘閱讀程式碼中的註解，然後儲存變更。

更新 Deployment YAML 檔案

部署資訊清單已經部分設定好環境變數、磁碟區掛載和探查。您只需要以您專屬的 ACR 端點更新容器映像參考。

  1. 開啟 k8s/deployment.yaml 檔案，並找出 image: <YOUR_ACR_ENDPOINT>/aks-config-api:latest 這一行。

  2. 將 <YOUR_ACR_ENDPOINT> 替換為您稍早在本練習中記錄的值。

  3. 花幾分鐘閱讀程式碼中的註解，然後儲存變更。

將資訊清單套用至 AKS

在本節中，您將把資訊清單套用至 AKS。以下步驟都在 VS Code 終端機中執行。執行命令之前，請確認您位於專案根目錄。

  1. 執行下列命令以套用 ConfigMap。

    kubectl apply -f k8s/configmap.yaml

  2. 執行下列命令以套用 Secrets。

    kubectl apply -f k8s/secrets.yaml

  3. 執行下列命令以套用 PersistentVolumeClaim。

    kubectl apply -f k8s/pvc.yaml

  4. 執行下列命令以套用 Deployment。

    kubectl apply -f k8s/deployment.yaml

  5. 執行下列命令以建立 Service。

    kubectl apply -f k8s/service.yaml

  6. 建立 Service 後，部署可能需要幾分鐘才能完成。下列命令會監視服務，並在可用時更新 pod 的外部 IP 位址。請記下此外部 IP 位址，稍後在本練習中會用到。IP 位址出現後，按下 ctrl + c 結束命令。

    kubectl get svc aks-config-api-service -w

執行用戶端應用程式

在本節中，您將設定 Python 環境，然後使用用戶端應用程式對 API 執行操作。

設定 Python 環境

在本節中，您將建立 Python 環境並安裝相依性。

  1. 請確認您在終端機中位於專案的 client 資料夾。

  2. 在 VS Code 終端機中執行下列命令以建立 Python 環境。

    python -m venv .venv

  3. 執行下列命令以啟用 Python 環境。Note: 在 Linux/macOS 上，請使用 Bash 命令；在 Windows 上，請使用 PowerShell 命令。若在 Windows 上使用 Git Bash，請使用 source .venv/Scripts/activate。

    Bash

    source .venv/bin/activate

    PowerShell

    .\.venv\Scripts\Activate.ps1

  4. 在 VS Code 終端機中執行下列命令以安裝相依性。

    pip install -r requirements.txt

  5. 在 client 目錄中建立一個 .env 檔案，並加入下列程式碼。請將 <API_IP_address> 替換為您稍早在本練習中複製的值。

    # API endpoint - update this with the external IP from the LoadBalancer service
# Get the IP with: kubectl get services
API_ENDPOINT=http://<API_IP_address>

使用應用程式執行操作

設定好 Python 環境並安裝相依性後，您現在可以執行用戶端應用程式來測試已部署的 API。API 會將所有操作記錄到持續性磁碟區，而用戶端則提供選單式介面，讓您與各種端點互動。

  1. 在終端機中執行下列命令以啟動主控台應用程式。如有需要，請參考本練習前面的命令先啟用環境，再執行此命令。

    python main.py

  2. 輸入 1 以啟動 Check API Health (Liveness) 選項。這會驗證 API 容器正在執行且會回應健康狀態檢查，這也是 Kubernetes liveness probe 使用的相同端點。請注意，其傳回的資訊包含在 ConfigMap 中設定的非敏感學生姓名。

    [*] Checking API health...
✓ API is healthy
  Service: aks-config-api
  Version: 1.0.0
  Student: YourNameHere

  3. 輸入 2 以啟動 Check API Readiness (Foundry Connectivity) 選項。這會確認 API 可以成功連線到 Foundry model endpoint，並已準備好處理推論要求。

  4. 輸入 3 以啟動 View Secrets Information 選項。此功能僅供您確認 secrets 已在 pod 中設定，僅用於示範目的。在輸出中您可以檢視 secrets 的相關資訊，輸出內容已遮罩處理。

    Secret Details:

  secret_endpoint:
    Loaded: True
    Value: SecretEndp...
    Length: 19 characters

  secret_access_key:
    Loaded: True
    Value: ***3456
    Length: 21 characters

  5. 輸入 5 以啟動 List All Products 選項。這會顯示 API 內含的模擬資料。

  6. 現在 API 已為數個不同端點記錄操作，是時候檢視記錄了。輸入 6 以啟動 View Log Summary 選項，查看不同操作的摘要。請注意要求總數，以及對 /readyz 與 /healthz 端點的要求數。這兩項操作會依據 deployment.yaml 檔案中設定的排程自動執行。

    ✓ Log summary retrieved

Total requests: 220
Student: YourNameHere

First request: 2025-12-21T02:19:48.953308
Last request: 2025-12-21T02:46:08.952772

Requests by endpoint:
  /readyz: 160
  /healthz: 56
  /secrets: 1
  /products: 1

  7. 您可以繼續產生記錄資訊，完成後輸入 7 以結束應用程式。

清理資源

現在您已完成本練習，應刪除您建立的雲端資源，以避免不必要的資源使用。

  1. 在 VS Code 終端機中執行下列命令，以刪除資源群組以及群組中的所有資源。請將 <rg-name> 替換為您稍早在本練習中選擇的名稱。此命令會在 Azure 中啟動背景工作以刪除資源群組。

    az group delete --name <rg-name> --no-wait --yes

CAUTION: 刪除資源群組會刪除其中包含的所有資源。如果您為本練習選擇的是現有資源群組，則本練習範圍之外的任何既有資源也會一併刪除。

疑難排解

如果您在完成本練習時遇到問題，請嘗試下列疑難排解步驟：

使用部署指令碼檢查部署狀態

  • 執行部署指令碼，並選取 5. Check deployment status 選項，以驗證所有已部署資源的狀態。
  • 此命令會檢查：
    • ACR 佈建狀態與就緒情況
    • AKS 叢集佈建狀態
    • Kubernetes 資源（ConfigMap、Secrets、PVC、Deployment 可用性、Service LoadBalancer IP）
  • 使用此輸出來識別可能造成問題的元件。

驗證 YAML 檔案完整性

  • 請確認所有 YAML 內容都已正確加入 configmap.yaml、secrets.yaml 與 pvc.yaml，且縮排正確。
  • 確認已在 deployment.yaml 中正確更新 ACR 端點（將 <YOUR_ACR_ENDPOINT> 替換為您實際的 ACR 端點）。
  • 如果您在初次部署後變更了任何 YAML 檔案，請使用 kubectl apply -f k8s/<filename>.yaml 重新套用該檔案。
  • 更新 ConfigMap 或 Secret 檔案後，請執行滾動重新啟動以重新載入設定：kubectl rollout restart deployment aks-config-api。

驗證用戶端設定

  • 請確認您已在 client 資料夾中建立 .env 檔案，並將 API_ENDPOINT 設定為 LoadBalancer 的外部 IP（例如：http://20.xxx.xxx.xxx）。
  • 在終端機中執行 curl http://<external-ip>/healthz，確認您可以連到 API 端點。

檢查 Python 環境與相依性

  • 在執行用戶端應用程式之前，確認虛擬環境已啟用。
  • 執行 pip list，確認 requirements.txt 中的所有套件都已成功安裝。
  • 請確認您是從 client 目錄執行用戶端。
