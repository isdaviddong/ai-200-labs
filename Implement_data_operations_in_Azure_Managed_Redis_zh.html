在 Azure Managed Redis 中執行資料作業

在本練習中，您將建立 Azure Managed Redis 資源，並建置一個 Python 主控台應用程式，使用 redis-py 函式庫來執行常見的資料作業。您會使用 Redis hash 資料結構來儲存與擷取鍵值組、以 Time-To-Live (TTL) 設定管理金鑰過期時間，並從快取中刪除金鑰。

本練習將執行的工作：

  • 下載專案起始檔案
  • 建立 Azure Managed Redis 資源
  • 將程式碼加入起始檔案以完成主控台應用程式
  • 執行主控台應用程式以執行資料作業

本練習約需 30 分鐘完成。

開始之前

若要完成本練習，您需要：

  • Azure 訂用帳戶。如果您尚未擁有，您可以 註冊一個。
  • Visual Studio Code 安裝於其中一個 支援的平台。
  • Python 3.12 或更新版本。
  • 最新版的 Azure CLI。
  • Azure CLI 的 redisenterprise 延伸模組。您可以執行 az extension add --name redisenterprise 命令來安裝它。

下載專案起始檔案並部署 Azure Managed Redis

在本節中，您將下載主控台應用程式的起始檔案，並使用指令碼初始化將 Azure Managed Redis 部署到您的訂用帳戶。Azure Managed Redis 的部署需要 5 到 10 分鐘才能完成。

  1. 開啟瀏覽器並輸入下列 URL 以下載起始檔案。該檔案會儲存在您的預設下載位置。

    https://github.com/MicrosoftLearning/mslearn-azure-ai/raw/main/downloads/python/amr-data-operations-python.zip

  2. 將該檔案複製或移動到系統中您想要進行專案作業的位置，然後將它解壓縮到資料夾中。

  3. 啟動 Visual Studio Code (VS Code)，並在選單中選取 File > Open Folder... ，然後選擇包含專案檔案的資料夾。

  4. 此專案同時包含 Bash（azdeploy.sh）與 PowerShell（azdeploy.ps1）的部署指令碼。開啟適合您環境的檔案，並變更指令碼頂端的兩個值以符合您的需求，然後儲存變更。 Note: 請勿變更指令碼中的其他任何內容。

    "<your-resource-group-name>" # Resource Group name
"<your-azure-region>" # Azure region for the resources

  5. 在功能表列中選取 Terminal > New Terminal ，以在 VS Code 中開啟終端機視窗。

  6. 執行下列命令登入您的 Azure 帳戶。依照提示選取要用於本練習的 Azure 帳戶與訂用帳戶。

    az login

  7. 執行下列命令，確認您的訂用帳戶具有 Azure Managed Redis 所需的資源提供者。

    az provider register --namespace Microsoft.Cache

  8. 執行下列命令以安裝 redisenterprise 的 Azure CLI 延伸模組。

    az extension add --name redisenterprise

  9. 在終端機中執行適當的命令以啟動指令碼。

    Bash

    bash azdeploy.sh

    PowerShell

    ./azdeploy.ps1

    Note: 如果 PowerShell 因指令碼未經數位簽章而封鎖它，請在同一個終端機工作階段中執行下列命令，然後再次執行部署指令碼。此命令只會變更目前 PowerShell 處理程序的執行原則。

    Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass

  10. 當指令碼執行時，輸入 1 以啟動 1. Create Azure Managed Redis resource 選項。

    此選項會在資源群組尚不存在時建立它，並開始部署 Azure Managed Redis。此程序會在 Azure 中以背景工作方式完成。

  11. 當主控台中出現下列訊息後，選取 Enter 以返回選單，然後選取 4 以結束指令碼。稍後您會再次執行此指令碼，以檢查部署狀態並建立 .env 專案所需的檔案。

    Azure Managed Redis 資源正在建立中，通常需要 5 到 10 分鐘才能完成。

    您可以在稍後的練習中從選單檢查部署狀態。

設定 Python 環境

在本節中，您將建立 Python 環境並安裝相依性。

  1. 在 VS Code 終端機中執行下列命令以建立 Python 環境。

    python -m venv .venv

  2. 執行下列命令以啟用 Python 環境。 Note: 在 Linux/macOS 上，請使用 Bash 命令。在 Windows 上，請使用 PowerShell 命令。如果是在 Windows 上使用 Git Bash，請使用 source .venv/Scripts/activate.

    Bash

    source .venv/bin/activate

    PowerShell

    .\.venv\Scripts\Activate.ps1

  3. 在 VS Code 終端機中執行下列命令以安裝相依性。

    pip install -r requirements.txt

完成應用程式

在本節中，您將程式碼加入 main.py 指令碼以完成主控台應用程式。您會在本練習稍後確認 Azure Managed Redis 資源已完全部署並建立 .env 檔案之後再執行此應用程式。

  1. 開啟 main.py 檔案以開始加入程式碼。

Note: 您加入到應用程式中的程式碼區塊應與該段程式碼的註解對齊。

加入用戶端連線

在本節中，您將加入程式碼，使用 redis-py 函式庫建立與 Azure Managed Redis 的連線。這段程式碼會從環境變數擷取連線認證，並建立已設定為安全 SSL 通訊的 Redis 用戶端執行個體。

  1. 尋找 # BEGIN CONNECTION CODE SECTION 註解，並在該註解下方加入下列程式碼。請務必確認程式碼縮排正確。

    try:
    # Azure Managed Redis with Non-Clustered policy uses standard Redis connection
    redis_host = os.getenv("REDIS_HOST")
    redis_key = os.getenv("REDIS_KEY")

    # Non-clustered policy uses standard Redis client connection
    r = redis.Redis(
        host=redis_host,
        port=10000,  # Azure Managed Redis uses port 10000
        ssl=True,
        decode_responses=True, # Decode responses to strings
        password=redis_key,
        socket_timeout=30,  # Add timeout for better reliability
        socket_connect_timeout=30,
    )

    print(f"Connected to Redis at {redis_host}")
    input("\nPress Enter to continue...")
    return r

加入用於儲存與擷取資料的程式碼

在本節中，您將加入程式碼，使用 hset 與 hgetall 命令來操作 Redis hash 資料結構。hset 方法會在單一金鑰下儲存多個欄位值配對，而 hgetall 則會擷取指定金鑰的所有欄位與值。

  1. 尋找 # BEGIN STORE AND RETRIEVE CODE SECTION 註解，並在該註解下方加入下列程式碼。請務必確認程式碼縮排正確。

    def store_hash_data(r, key, value) -> None:
    """Store hash data in Redis"""
    clear_screen()
    print(f"Storing hash data for key: {key}")
    result = r.hset(key, mapping=value) # Store hash data
    if result > 0: # New fields were added
        print(f"Data stored successfully under key '{key}' ({result} new fields added)")
    else:
        print(f"Data updated successfully under key '{key}' (all fields already existed)")
    input("\nPress Enter to continue...")

def retrieve_hash_data(r, key) -> None:
    """Retrieve hash data from Redis"""
    clear_screen()
    print(f"Retrieving hash data for key: {key}")
    retrieved_value = r.hgetall(key) # Retrieve hash data
    if retrieved_value:
        print("\nRetrieved hash data:")
        for field, value in retrieved_value.items():
            print(f"  {field}: {value}")
    else:
        print(f"Key '{key}' does not exist.")

    input("\nPress Enter to continue...")

加入設定與擷取到期時間的程式碼

在本節中，您將加入程式碼，使用 expire 與 ttl 命令來管理金鑰到期時間。expire 方法會在金鑰上設定 Time-To-Live (TTL)，使其在指定秒數後自動過期，而 ttl 則會擷取金鑰過期前的剩餘時間。

  1. 尋找 # BEGIN EXPIRATION CODE SECTION 註解，並在該註解下方加入下列程式碼。請務必確認程式碼縮排正確。

    def set_expiration(r, key) -> None:
    """Set an expiration time for a key"""
    clear_screen()
    print("Set expiration time for a key")
    # Set expiration time, 1 hour equals 3600 seconds
    expiration = int(input("Enter expiration time in seconds (default 3600): ") or 3600)
    result = r.expire(key, expiration) # Set expiration time
    if result:
        print(f"Expiration time of {expiration} seconds set for key '{key}'")
    else:
        print(f"Key '{key}' does not exist. Expiration not set.")

    input("\nPress Enter to continue...")

def retrieve_expiration(r, key) -> None:
    """Retrieve the TTL of a key"""
    clear_screen()
    print(f"Retrieving the current TTL of {key}...")
    ttl = r.ttl(key) # Get current TTL
    if ttl == -2: # Key does not exist
        print(f"\nKey '{key}' does not exist.")
    elif ttl == -1: # No expiration set
        print(f"\nKey '{key}' has no expiration set (persists indefinitely).")
    else:
        print(f"\nCurrent TTL for '{key}': {ttl} seconds")
    input("\nPress Enter to continue...")

加入刪除資料的程式碼

在本節中，您將加入程式碼，使用 delete 命令從 Redis 移除金鑰。 delete 方法會從快取中永久移除金鑰及其相關值，釋放記憶體並確保資料不再可存取。

  1. 尋找 # BEGIN DELETE CODE SECTION 註解，並在該註解下方加入下列程式碼。請務必確認程式碼縮排正確。

    def delete_key(r, key) -> None:
    """Delete a key"""
    clear_screen()
    print(f"Deleting key: {key}...")
    result = r.delete(key) # Delete the key
    if result == 1:
        print(f"Key '{key}' deleted successfully.")
    else:
        print(f"Key '{key}' does not exist.")
    input("\nPress Enter to continue...")

  2. 將您的變更儲存到 main.py 檔案。

驗證資源部署

在本節中，您將再次執行部署指令碼，以確認 Azure Managed Redis 部署是否完成，並建立 .env 檔案，其中包含端點與存取金鑰值。

  1. 在終端機中執行適當的命令以啟動部署指令碼。如果您已關閉先前的終端機，請選取 Terminal > New Terminal 以在選單中開啟新的終端機。

    Bash

    bash azdeploy.sh

    PowerShell

    ./azdeploy.ps1

    Note: 如果 PowerShell 因指令碼未經數位簽章而封鎖它，請在同一個終端機工作階段中執行下列命令，然後再次執行部署指令碼。此命令只會變更目前 PowerShell 處理程序的執行原則。

    Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass

  2. 當部署選單出現時，輸入 2 以執行 2. Check deployment status 選項。如果狀態顯示為 Succeeded，請繼續下一步。若否，請等待幾分鐘後再試一次此選項。

  3. 部署完成後，輸入 3 以執行 3. Create database and retrieve endpoint and access key 選項。這會建立資料庫、啟用存取金鑰驗證，並擷取端點與存取金鑰。接著它會建立包含這些值的 .env 檔案。

  4. 檢閱 .env 檔案以確認值已存在，然後輸入 4 以結束部署指令碼。

執行主控台應用程式

在本節中，您將執行已完成的主控台應用程式，以執行各種 Redis 資料作業。此應用程式提供選單式介面，讓您可以儲存 hash 資料、擷取值、管理金鑰過期時間，以及刪除金鑰。

  1. 在終端機中執行下列命令以啟動主控台應用程式。如有需要，請參考稍早練習中的命令先啟用環境，再執行此命令。

    python main.py

  2. 應用程式提供下列選項。請選取 1. Store hash data 開始操作。

    1. Store hash data
2. Retrieve hash data
3. Set expiration
4. Retrieve expiration (TTL)
5. Delete key
6. Exit

  3. 接著依序選取其餘選項，以執行不同作業。

Note: 您可以依自己選擇的任意順序執行這些選項。例如，在儲存 hash 資料後，您可以擷取到期資訊，以了解該金鑰未設定到期時間。

應用程式中使用的模擬 hash 資料定義於 main() 函式的開頭。您可以更新程式碼以使用不同的金鑰，或向 hash 資料新增更多值。

清除資源

既然您已完成本練習，應刪除所建立的雲端資源，以避免不必要的資源使用。

  1. 在 VS Code 終端機中執行下列命令以刪除資源群組及群組中的所有資源。請將 <rg-name> 替換為您稍早在練習中選擇的名稱。此命令會在 Azure 中啟動背景工作以刪除資源群組。

    az group delete --name <rg-name> --no-wait --yes

CAUTION: 刪除資源群組會刪除其中包含的所有資源。如果您為本練習選用了現有的資源群組，則此練習範圍外的任何既有資源也會一併刪除。

疑難排解

如果您在完成本練習時遇到問題，請嘗試下列疑難排解步驟：

驗證 Azure Managed Redis 資源部署

  • 瀏覽至 Azure portal 並找出您的資源群組。
  • 確認 Azure Managed Redis 資源顯示的 Provisioning State 為 Succeeded。
  • 確認資源已啟用 Public network access，且 Access keys authentication 設為 Enabled。

檢查程式碼完整性與縮排

  • 確認所有程式碼區塊都已加入 main.py 中對應的 BEGIN/END 註解標記之間。
  • 確認 Python 縮排一致（使用空格，不使用 Tab），且所有程式碼在函式內都正確對齊。
  • 確認在指定區段之外沒有任何程式碼被意外刪除或修改。

驗證環境變數

  • 確認 .env 檔案存在於專案資料夾中，且包含有效的 REDIS_HOST 與 REDIS_KEY 值。
  • 確認 .env 檔案與 main.py 位於相同目錄中。

檢查 Python 環境與相依性

  • 確認在執行應用程式前已啟用虛擬環境。
  • 確認 requirements.txt 中的所有套件都已成功安裝，可執行 pip list。
