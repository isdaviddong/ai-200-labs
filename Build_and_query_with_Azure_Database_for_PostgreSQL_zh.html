在 Azure Database for PostgreSQL 上建置代理工具後端

在本練習中，您將建立一個 Azure Database for PostgreSQL 充當 tool 後端 AI 代理人。資料庫儲存代理程式在操作期間可以讀取和寫入的對話上下文和任務狀態。您為代理記憶體設計一個模式，構建 Python 作為代理工具的功能，並測試完整的工作流程。該模式為構建提供了基礎 AI 跨會話維護持久性記憶體並可以恢復中斷的任務的代理程式。

本練習執行的工作：

  • 下載專案啟動檔案並設定部署腳本
  • 部署一個 Azure Database for PostgreSQL Flexible Server 和 Microsoft Entra 驗證
  • 建造 Python tool 對話和任務狀態管理功能
  • 使用對話、訊息和任務檢查點表為代理記憶體建立資料庫結構描述
  • 使用提供的測試腳本測試代理記憶體工作流程
  • 使用查詢對話上下文 SQL

本練習大約需要 30 分鐘即可完成。

開始之前

要完成練習，您需要：

  • 具有部署必要 Azure 服務權限的 Azure 訂閱。如果您還沒有，您可以 註冊一個.
  • Visual Studio Code 在其支援的平台之一上 支援的平台.
  • 最新版本的 Azure CLI.
  • Python 3.12 或更新版本。
  • PostgreSQL 命令列工具 (psql)

完成 tool 功能應用程式

在本節中，您將完成 agent_tools.py 文件透過添加功能 AI 代理可以呼叫來持久化和擷取狀態。這些函數充當代理與資料庫的介面。這 test_workflow.py 您稍後將在本練習中執行的腳本匯入這些函數以示範代理程式如何使用它們。

  1. 打開 agent-backend/agent_tools.py 文件輸入 VS Code.

  2. 搜尋 BEGIN CREATE CONVERSATION FUNCTION 註釋並直接在註釋後面添加以下程式碼。此函數會建立一個具有唯一會話的新對話記錄 ID 和商店可選 metadata 作為 JSONB.

    def create_conversation(user_id: str, metadata: dict = None) -> dict:
    """Create a new conversation and return its details."""
    session_id = uuid.uuid4()
    with get_connection() as conn:
        with conn.cursor() as cur:
            cur.execute(
                """
                INSERT INTO conversations (session_id, user_id, metadata)
                VALUES (%s, %s, %s)
                RETURNING id, session_id, started_at
                """,
                (str(session_id), user_id, psycopg.types.json.Json(metadata or {}))
            )
            row = cur.fetchone()
            conn.commit()
            return {
                "conversation_id": row[0],
                "session_id": str(row[1]),
                "started_at": row[2].isoformat()
            }

  3. 搜尋 BEGIN RETRIEVE CONVERSATION HISTORY FUNCTION 註釋並直接在註釋後面添加以下程式碼。此函數從對話中擷取訊息，按時間順序排列。

    def get_conversation_history(conversation_id: int, limit: int = 50) -> list:
    """Retrieve recent messages from a conversation."""
    with get_connection() as conn:
        with conn.cursor() as cur:
            cur.execute(
                """
                SELECT id, role, content, created_at, metadata
                FROM messages
                WHERE conversation_id = %s
                ORDER BY created_at DESC
                LIMIT %s
                """,
                (conversation_id, limit)
            )
            rows = cur.fetchall()
            return [
                {
                    "id": row[0],
                    "role": row[1],
                    "content": row[2],
                    "created_at": row[3].isoformat(),
                    "metadata": row[4]
                }
                for row in reversed(rows)  # Return in chronological order
            ]

  4. 搜尋 BEGIN TASK CHECKPOINT FUNCTIONS 註釋並直接在註釋後面添加以下程式碼。此函數使用 upsert 模式來保存或更新任務狀態，允許代理程式恢復中斷的任務。

    def save_task_state(conversation_id: int, task_name: str, status: str, checkpoint_data: dict) -> dict:
    """Save or update a task checkpoint."""
    with get_connection() as conn:
        with conn.cursor() as cur:
            cur.execute(
                """
                INSERT INTO task_checkpoints (conversation_id, task_name, status, checkpoint_data)
                VALUES (%s, %s, %s, %s)
                ON CONFLICT (conversation_id, task_name)
                DO UPDATE SET
                    status = EXCLUDED.status,
                    checkpoint_data = EXCLUDED.checkpoint_data,
                    updated_at = CURRENT_TIMESTAMP
                RETURNING id, updated_at
                """,
                (conversation_id, task_name, status, psycopg.types.json.Json(checkpoint_data))
            )
            row = cur.fetchone()
            conn.commit()
            return {
                "checkpoint_id": row[0],
                "updated_at": row[1].isoformat()
            }

  5. 將您的變更儲存到 agent_tools.py file.

  6. 花幾分鐘查看應用程式中的所有程式碼。

接下來，您建立架構來支援代理程式。

使用以下命令建立代理記憶體架構 psql

在本節中，您將連接到 PostgreSQL 伺服器使用 psql 命令列 tool 並為代理記憶體創建資料庫結構描述。該架構包括三個表：一張用於對話（代理會話），一個用於這些對話中的消息，一個用於使代理能夠恢復中斷的工作的任務檢查點。

  1. 執行以下命令使用環境變數連接到伺服器。這 PGPASSWORD 環境變數自動用於身份驗證。

    Bash

    psql "host=$DB_HOST port=5432 dbname=$DB_NAME user=$DB_USER sslmode=require"

    PowerShell

    psql "host=$env:DB_HOST port=5432 dbname=$env:DB_NAME user=$env:DB_USER sslmode=require"

  2. 運行以下命令透過檢查來驗證連接 PostgreSQL 版本。

    SELECT version();

  3. 執行以下命令為代理後端建立資料庫。這 \c 命令連接到新資料庫。

    CREATE DATABASE agent_memory;
\c agent_memory

  4. 執行以下命令來建立對話表（代理會話）。此表存儲會話 metadata 並將訊息連結到特定對話。

    CREATE TABLE conversations (
    id BIGSERIAL PRIMARY KEY,
    session_id UUID NOT NULL UNIQUE,
    user_id VARCHAR(255) NOT NULL,
    started_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    ended_at TIMESTAMP WITH TIME ZONE,
    metadata JSONB DEFAULT '{}'::jsonb
);

  5. 執行以下命令為對話中的消息建立表。此表儲存角色（user, assistant, system， 或者 tool）和每個訊息的內容。

    CREATE TABLE messages (
    id BIGSERIAL PRIMARY KEY,
    conversation_id BIGINT NOT NULL REFERENCES conversations(id) ON DELETE CASCADE,
    role VARCHAR(50) NOT NULL CHECK (role IN ('user', 'assistant', 'system', 'tool')),
    content TEXT NOT NULL,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    metadata JSONB DEFAULT '{}'::jsonb
);

  6. 執行以下命令建立任務檢查點表。此表支援代理狀態持久化，以便代理可以恢復中斷的任務。

    CREATE TABLE task_checkpoints (
    id BIGSERIAL PRIMARY KEY,
    conversation_id BIGINT REFERENCES conversations(id) ON DELETE CASCADE,
    task_name VARCHAR(255) NOT NULL,
    status VARCHAR(50) NOT NULL CHECK (status IN ('pending', 'in_progress', 'completed', 'failed')),
    checkpoint_data JSONB NOT NULL DEFAULT '{}'::jsonb,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

  7. 執行以下命令以建立優化常見查詢的索引。這些索引提高了透過對話或時間戳記擷取訊息時的效能。

    CREATE INDEX idx_messages_conversation_id ON messages(conversation_id);
CREATE INDEX idx_messages_created_at ON messages(created_at);
CREATE INDEX idx_task_checkpoints_conversation_id ON task_checkpoints(conversation_id);
CREATE INDEX idx_conversations_session_id ON conversations(session_id);

  8. 您的應用程式 completed 在練習的早期使用 ON CONFLICT，這需要唯一的約束。在您的中運行以下命令 psql 在終端中添加它。

    ALTER TABLE task_checkpoints
ADD CONSTRAINT unique_conversation_task
UNIQUE (conversation_id, task_name);

  9. 執行以下命令以驗證架構是否已正確建立。

    \dt

    您應該看到列出的三個表。

  10. 輸入 exit 關閉 psql 會話並返回到終端。

  1. 運行以下命令導航到 agent-backend 目錄。

    cd agent-backend

  2. 執行以下命令建立虛擬環境 test_workflow.py 應用程式.根據您的環境，命令可能是 python 或者 python3.

    python -m venv .venv

  3. 運行以下命令來激活 Python 環境。 注意： 在 Linux/macOS 上，使用 Bash 命令。在 Windows 上，使用 PowerShell 命令。如果使用 Git Bash 在 Windows 上，使用 source .venv/Scripts/activate.

    PowerShell

查詢對話上下文

在本節中，您將練習查詢代理程式用於做出決策的資料。

  1. 運行以下命令連接到 agent_memory 使用環境變數的資料庫。

    Bash

    psql "host=$DB_HOST port=5432 dbname=agent_memory user=$DB_USER sslmode=require"

    PowerShell

    psql "host=$env:DB_HOST port=5432 dbname=agent_memory user=$env:DB_USER sslmode=require"

    提示： 當查詢結果顯示時， psql 如果目前終端機視窗無法容納結果，則使用尋呼機。如果是這樣，請按 q 結束尋呼機並返回 psql 迅速的。最大化終端機視窗將減少這種情況的發生，並且可以更輕鬆地查看命令的結果。

  2. 執行以下查詢來尋找特定的所有對話 user。測試腳本創建了一個對話 user_id 設定為 user_123.

    SELECT id, session_id, started_at, metadata
FROM conversations
WHERE user_id = 'user_123'
ORDER BY started_at DESC;

  3. 執行以下查詢以取得所有對話中的最新消息。這將傳回測試腳本儲存的訊息。

    SELECT c.session_id, m.role, m.content, m.created_at
FROM messages m
JOIN conversations c ON m.conversation_id = c.id
ORDER BY m.created_at DESC
LIMIT 10;

  4. 執行以下查詢來查找 completed 任務。測試腳本將任務狀態更新為 completed 在工作流程結束時。

    SELECT
    c.session_id,
    t.task_name,
    t.status,
    t.checkpoint_data,
    t.updated_at
FROM task_checkpoints t
JOIN conversations c ON t.conversation_id = c.id
WHERE t.status = 'completed';

  5. 執行以下查詢以按每個對話中的角色對訊息進行計數。這有助於理解分佈 user, assistant, system， 和 tool 消息。

    SELECT
    c.id AS conversation_id,
    m.role,
    COUNT(*) AS message_count
FROM conversations c
JOIN messages m ON c.id = m.conversation_id
GROUP BY c.id, m.role
ORDER BY c.id, m.role;

  6. 輸入 quit 在 psql 提示結束。

現在您已完成練習，您應該刪除您建立的雲端資源，以避免不必要的資源使用。

  1. 在中運行以下命令 VS Code 終端刪除資源組以及該組中的所有資源。代替 <rg-name> 與您之前在練習中選擇的名稱。該命令將在 Azure 中啟動背景任務以刪除資源組。

疑難排解

如果您在此練習中遇到問題，請嘗試以下步驟：

psql connection fails

  • 確保 .env 文件是透過運行部署腳本選項創建的 4
  • 確保你跑了 source .env (Bash） 或者 . ..env.ps1 (PowerShell) 載入環境變數
  • 訪問令牌大約一小時後過期；運行部署腳本選項 4 再次產生新的令牌
  • 透過執行部署腳本選項來驗證伺服器是否已準備就緒 3

Access denied or authentication errors

  • 確保 Microsoft Entra 透過執行部署腳本選項設定管理員 2
  • 核實 PGPASSWORD 在您的終端會話中設定正確
  • 確保您使用的是正確的 DB_USER 值（您的 Azure 帳戶 電子郵件）

Python test script fails

  • 確保 Python 虛擬環境已啟動（您應該看到 (.venv) 在你的終端機提示中）
  • 確保安裝了依賴項： pip install -r requirements.txt
  • 確保您創建了 agent_memory 資料庫和所有表 psql
  • 確保您新增了唯一約束 task_checkpoints 桌子

Database or table not found errors

  • 確保您已連接到 agent_memory 資料庫使用 \c agent_memory 在 psql
  • 透過運行驗證表是否存在 \dt 在 psql
  • 重新運行 CREATE TABLE 如果表格遺失的語句

Python venv activation issues

  • 在 Linux/macOS 上，使用： source .venv/bin/activate
  • 在 Windows 上 PowerShell， 使用： ..venv\Scripts\Activate.ps1
  • 如果 activate 腳本遺失，重新安裝 python3-venv 打包並重新建立 venv
