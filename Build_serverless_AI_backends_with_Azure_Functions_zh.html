使用 Azure Functions 建立 MCP 伺服器

Model Context Protocol (MCP) 是一種開放標準，用來定義 AI 代理與語言模型如何探索並呼叫外部工具。Azure Functions 包含 MCP extension，可讓您將 function apps 公開為 MCP 伺服器，其中每個函式都會成為 MCP 用戶端可呼叫的工具。

在此練習中，您將建立使用 MCP extension 的 Azure Functions 專案、為文件處理定義工具觸發函式、設定 MCP 伺服器設定，並透過 GitHub Copilot 的 agent mode 在本機連線測試該伺服器。

注意： 此練習使用 Azure Functions MCP extension，而此 extension 仍在快速演進中。請造訪 Azure Functions MCP extension documentation 以取得最新的設定指示、API 介面與組態選項。

此練習中執行的工作：

  • 建立使用 MCP extension 的新 Azure Functions 專案
  • 在 host.json
  • 於 function_app.py
  • 驗證 Python 環境
  • 使用 GitHub Copilot 的 agent mode 在本機測試 MCP 伺服器

此練習大約需要 25 分鐘完成。

開始之前

若要完成本練習，您需要：

  • Visual Studio Code 安裝於其中一個 支援的平台.
  • The Azure Functions extension for Visual Studio Code.
  • Azure Functions Core Tools v4 or later.
  • Python 3.9 或更新版本。
  • The GitHub Copilot extension for Visual Studio Code.

建立使用 MCP extension 的新 Functions 專案

在本節中，您將在 Visual Studio Code 中使用 Azure Functions extension 建立新的 Azure Functions 專案，並在 host.json中設定 MCP 伺服器設定。此 extension 會建立專案結構、產生 .vscode 的整合式偵錯設定檔，並設定 Python v2 程式設計模型。

  1. 為專案建立資料夾（例如 mcp-server-functions），並在 Visual Studio Code 中透過選取 File > Open Folder... 於選單中開啟它。

  2. 按下 Ctrl+Shift+P 以開啟 Command Palette。執行 Azure Functions: Create Function... 命令，並在提示時選擇下列選項：

    選項 動作
    Select the folder... 選取您在前一步開啟的資料夾
    選取專案類型 Select Python
    Select a Python interpreter... Select python 3.12
    Select a template... Select HTTP trigger
    函式名稱 接受預設值 http_trigger
    授權層級 Select ANONYMOUS

    此 extension 會建立專案結構，其中包括 function_app.py, host.json, local.settings.json, requirements.txt，以及 .vscode 資料夾與其中的 launch.json, tasks.json, and extensions.json。在下一節中，會取代 function_app.py 內由 scaffold 建立的 HTTP trigger 函式。

  3. Open host.json 並以下列程式碼取代其內容，然後儲存您的變更。 extensionBundle 必須使用 Preview bundle，因為 mcpToolTrigger binding type 是預覽功能，不包含在穩定版 extension bundle 中。 mcpToolTrigger 區段會定義 MCP 伺服器名稱、版本，以及 MCP 用戶端連線時顯示的說明。

    {
    "version": "2.0",
    "extensionBundle": {
        "id": "Microsoft.Azure.Functions.ExtensionBundle.Preview",
        "version": "[4.*, 5.0.0)"
    },
    "extensions": {
        "mcpToolTrigger": {
            "serverName": "document-tools",
            "serverVersion": "1.0.0",
            "serverInstructions": "Tools for document processing and classification"
        }
    }
}

  4. Open local.settings.json 並以下列程式碼取代其內容。

    {
  "IsEncrypted": false,
  "Values": {
    "AzureWebJobsStorage": "",
    "FUNCTIONS_WORKER_RUNTIME": "python",
    "AzureWebJobsSecretStorageType": "Files"
  }
}

  5. 建立 .vscode/mcp.json 檔案於專案中，以向 Visual Studio Code 註冊本機 MCP 端點。此檔案會告知 Visual Studio Code 要在哪裡找到 MCP 伺服器，以及如何連線到它們。將下列程式碼加入該檔案，然後儲存您的變更。

    {
    "servers": {
        "document-tools-local": {
            "type": "sse",
            "url": "http://localhost:7071/runtime/webhooks/mcp/sse"
        }
    }
}

定義 MCP 工具觸發函式

在本節中，您將定義兩個 MCP 工具觸發函式，讓 MCP 用戶端可探索並使用。每個函式都會使用 @app.generic_trigger() 裝飾器搭配 mcpToolTrigger 類型。您會透過觸發器設定來定義工具名稱、描述與輸入屬性，而函式會接收來自已連線 MCP 用戶端的工具呼叫要求。

  1. Open function_app.py 於 Visual Studio Code Explorer 側邊欄中，並以下列程式碼取代其內容，以定義兩個 MCP 工具觸發函式：

    import azure.functions as func
import json
import logging

# Initialize the FunctionApp instance that registers all trigger functions
app = func.FunctionApp()

# Define an MCP tool trigger that exposes "summarize_text" to MCP clients.
# toolProperties defines the input schema: a single "text" string parameter.
@app.generic_trigger(
    arg_name="context",
    type="mcpToolTrigger",
    toolName="summarize_text",
    description="Summarize a block of text into key points",
    toolProperties='[{"propertyName": "text", "propertyType": "string", "description": "The text to summarize"}]'
)
def summarize_text(context: str) -> str:
    # Log the raw payload for debugging
    logging.info(f"summarize_text raw context: {context}")
    # Parse the outer request envelope sent by the MCP client
    request = json.loads(context)
    logging.info(f"summarize_text parsed request: {request}")
    # Extract the tool arguments from the request
    arguments = request.get("arguments", {})
    # Retrieve the "text" property defined in toolProperties
    text = arguments.get("text", "")

    # In a real implementation, call an Azure AI service here
    summary = f"Summary of {len(text.split())} words: {text[:100]}..."

    # Return a JSON response; the "content" field is displayed to the MCP client
    return json.dumps({"content": summary})

# Define a second MCP tool trigger that exposes "classify_document" to MCP clients.
# toolProperties defines two input parameters: "text" and "categories".
@app.generic_trigger(
    arg_name="context",
    type="mcpToolTrigger",
    toolName="classify_document",
    description="Classify a document into a category",
    toolProperties='[{"propertyName": "text", "propertyType": "string", "description": "The document text to classify"}, {"propertyName": "categories", "propertyType": "string", "description": "Comma-separated list of possible categories"}]'
)
def classify_document(context: str) -> str:
    # Log the raw payload for debugging
    logging.info(f"classify_document raw context: {context}")
    # Parse the outer request envelope sent by the MCP client
    request = json.loads(context)
    logging.info(f"classify_document parsed request: {request}")
    # Extract the tool arguments from the request
    arguments = request.get("arguments", {})
    # Retrieve the "text" and "categories" properties defined in toolProperties
    text = arguments.get("text", "")
    categories = arguments.get("categories", "general")

    # In a real implementation, call an Azure AI service here
    # Split the comma-separated categories string into a list
    category_list = [c.strip() for c in categories.split(",")]
    # Select the first category as the classification result
    selected_category = category_list[0] if category_list else "unknown"

    # Return a JSON response with the classification result
    return json.dumps({
        "content": f"Classification: {selected_category}",
        "category": selected_category
    })

  2. 儲存檔案並花幾分鐘檢閱程式碼。每個函式都會使用 @app.generic_trigger() 裝飾器搭配 mcpToolTrigger 類型。 toolName 會顯示在 MCP 用戶端的工具清單中，而 description 可協助語言模型了解何時使用各個工具。 toolProperties 參數會將輸入結構描述定義為屬性定義的 JSON 陣列。

驗證 Python 環境

在本節中，您將確認 Visual Studio Code 正在使用 Azure Functions extension 於專案設定期間建立之虛擬環境中的 Python 解譯器。

  1. 按下 Ctrl+Shift+P to open the Command Palette and run the Python: Select Interpreter 命令。請選取位於 .venv 資料夾中的解譯器（例如 ./.venv/bin/python）。這可確保當您使用 F5.

在本機測試 MCP 伺服器

在本節中，您將啟動本機 Functions runtime，並透過 GitHub Copilot 的 agent mode 連線到 MCP 伺服器，以驗證工具可被探索並傳回預期結果。

  1. 按下 F5 以在附加偵錯工具的情況下啟動 Functions runtime。若您收到有關必要 storage account 的警告，請選取 Skip for now 。Visual Studio Code 會啟動 Core Tools、附加偵錯工具，並開啟顯示函式端點的終端機面板。

    注意： 您也可以透過執行下列命令，在不附加偵錯工具的情況下啟動 runtime： func start in the integrated terminal.

    終端機輸出會顯示已註冊的 MCP 工具觸發函式。請確認您看到類似下列的輸出：

    Functions:
    classify_document: mcpToolTrigger
    summarize_text: mcpToolTrigger

    若兩個函式都出現，表示 MCP 伺服器正在執行並已可供連線。

  2. Visual Studio Code 會偵測 .vscode/mcp.json 檔案，並連線到 MCP 伺服器。開啟 GitHub Copilot chat 並切換到 Agent 模式。選取工具圖示（扳手），並尋找 document-tools-local 群組——這與 .vscode/mcp.json中的伺服器鍵名稱相符。請確認 summarize_text and classify_document 都顯示在該群組下方，並附有其描述。

使用明確提示進行測試

直接點名工具的明確提示是觸發 MCP 工具最可靠的方式。模型通常會呼叫該工具，但仍可能在回應中重新表述或摘要工具的原始輸出。若工具未被呼叫，請檢查終端機輸出以確認，再嘗試重新提交提示。

  1. 測試 classify_document 工具，方法是在 Copilot chat 中輸入下列提示。 注意： 當 Copilot 第一次呼叫 MCP 工具時，您可能會看到權限提示。請選取 Allow 以允許 Copilot 呼叫該工具。

    Use the classify_document tool to classify this text: 'This agreement is entered into by Party A and Party B'  with categories: contract, invoice, memo

    Copilot 會呼叫該工具並傳回回應。請確認結果包含 contract的分類。此 stub 實作會從以逗號分隔的清單中選取第一個類別，因此結果會符合您在提示中提供的第一個類別。請檢查終端機輸出中的函式呼叫記錄項目。

  2. 測試 summarize_text 工具，方法是在 Copilot chat 中輸入下列提示。

    Use the summarize_text tool to summarize this text: 'Azure Functions is a serverless compute service that lets you run event-triggered code without having to explicitly provision or manage infrastructure.'

    請確認結果包含以 Summary of 開頭的摘要字串，後面接著字數與輸入文字的截斷預覽。

使用自然語言提示進行測試

自然語言提示較可能直接回答問題而不呼叫工具，因為模型必須自行判斷某個工具是否相關。請檢查終端機輸出中的記錄項目，以確認工具是否真的被呼叫。若未呼叫，請嘗試重新表述提示或明確點名工具。

  1. 在 Copilot chat 中輸入下列提示：

    Is the following text an invoice, contract, or memo?
'Invoice B1234 for services rendered in January 2026. Total amount due: $5,000.'

    Copilot 應辨識出 classify_document 工具符合此要求並自動呼叫它。請確認結果會傳回分類，並檢查終端機輸出以確認函式已被呼叫。

  2. 輸入下列提示，以測試 summarize_text 工具的自然探索：

    Give me a brief summary of this text: 'Machine learning models require large datasets for training. The quality of the training data directly impacts model accuracy. Data preprocessing steps include cleaning, normalization, and feature extraction.'

    Copilot should invoke the summarize_text 工具並傳回摘要。請確認終端機輸出顯示函式呼叫。

  3. 按下 Shift+F5 以停止偵錯工具並關閉 Functions runtime。

後續步驟

在實際生產情境中，您會使用 Flex Consumption 方案將 function app 部署到 Azure，並使用 mcp_extension system key 驗證 MCP 用戶端連線，並將每個工具函式中的預留位置邏輯替換為使用 DefaultAzureCredential 以及 function app 的 managed identity 呼叫 Azure AI 服務的實作。詳細資訊請參閱 Azure Functions MCP extension documentation.

疑難排解

如果您在完成此練習時遇到問題，請嘗試下列疑難排解步驟：

Azure Functions Core Tools 無法啟動

  • 請透過執行下列命令確認已安裝 Azure Functions Core Tools v4 或更新版本： func --version （於終端機中）。
  • 請確認連接埠 7071 未被其他程序占用。
  • 請確認 local.settings.json 存在於專案根目錄。Azure Functions extension 會在專案建立期間產生此檔案。

MCP 工具未出現在 Copilot 中

  • Verify that .vscode/mcp.json 已儲存，且 URL 與本機端點相符（http://localhost:7071/runtime/webhooks/mcp/sse）。
  • 請確認 Functions runtime 正在執行，且終端機輸出中顯示兩個工具觸發函式。
  • 如果儲存檔案後仍未偵測到 MCP 伺服器設定，請嘗試重新啟動 Visual Studio Code。

函式呼叫傳回錯誤

  • 請檢查終端機輸出中的 Python 例外狀況或堆疊追蹤。
  • Verify function_app.py 的縮排正確，且所有程式碼都已依照範例輸入。
  • 請確認虛擬環境已啟用，且來自 requirements.txt 的所有相依性都已安裝。

檢查 Python 環境與相依性

  • 請確認 Visual Studio Code 正在使用來自 .venv 資料夾的 Python 解譯器，方法是執行 Python: Select Interpreter command.
  • 請確認來自 requirements.txt 的所有套件都已透過執行 pip list in the integrated terminal.
