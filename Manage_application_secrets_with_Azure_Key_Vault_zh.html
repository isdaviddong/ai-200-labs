使用 Azure Key Vault 管理秘密

AI 應用程式通常仰賴敏感認證，例如 API 金鑰、連線字串與憑證，才能存取模型端點和資料存放區。Azure Key Vault 為這些秘密提供集中且安全的儲存位置，具備 RBAC 存取控制、自動版本管理與稽核記錄，讓應用程式無需將認證內嵌在程式碼或組態檔中。

在本練習中，您將部署一個預先載入範例秘密的 Azure Key Vault，並建置一個使用 Azure SDK 展示核心秘密管理模式的 Python Flask Web 應用程式。您會擷取秘密並檢視其中繼資料、在不暴露值的情況下列出所有秘密屬性、建立新的秘密版本以模擬認證輪替，以及實作以時間為基礎的快取來減少 Key Vault API 呼叫。

本練習將執行的工作：

  • 下載專案起始檔
  • 建立 Azure Key Vault 並儲存範例秘密
  • 在起始檔中加入程式碼以完成應用程式
  • 執行應用程式以進行秘密作業

本練習大約需要 20 分鐘完成。

開始之前

若要完成本練習，您需要：

  • Azure 訂用帳戶。如果您還沒有，可以 註冊一個。
  • Visual Studio Code 安裝於其中一個 支援平台。
  • Python 3.12 或更新版本。
  • 最新版的 Azure CLI。

下載專案起始檔並部署 Azure Key Vault

在本節中，您將下載應用程式的起始檔，並使用指令碼將包含範例秘密的 Azure Key Vault 部署到您的訂用帳戶。

  1. 開啟瀏覽器並輸入下列 URL 以下載起始檔。檔案會儲存在您的預設下載位置。

    https://github.com/MicrosoftLearning/mslearn-azure-ai/raw/main/downloads/python/key-vault-python.zip

  2. 將檔案複製或移動到您系統中要進行專案作業的位置，然後將檔案解壓縮到資料夾中。

  3. 啟動 Visual Studio Code (VS Code)，然後在功能表中選取 File > Open Folder...，接著選擇包含專案檔案的資料夾。

  4. 此專案同時包含 Bash (azdeploy.sh) 與 PowerShell (azdeploy.ps1) 的部署指令碼。請開啟符合您環境的檔案，修改指令碼頂端的兩個值以符合您的需求，然後儲存變更。注意： 請勿變更指令碼中的其他任何內容。

    "<your-resource-group-name>" # Resource Group name
"<your-azure-region>" # Azure region for the resources

  5. 在功能表列中選取 Terminal > New Terminal，以在 VS Code 中開啟終端機視窗。

  6. 執行下列命令登入您的 Azure 帳戶。依照提示選取本練習要使用的 Azure 帳戶與訂用帳戶。

    az login

  7. 執行下列命令，確認您的訂用帳戶具有本練習所需的資源提供者。

    az provider register --namespace Microsoft.KeyVault

  8. 在終端機中執行適當的命令以啟動指令碼。

    Bash

    bash azdeploy.sh

    PowerShell

    ./azdeploy.ps1

    Note: 如果 PowerShell 因指令碼未經數位簽署而封鎖它，請在相同的終端機工作階段中執行下列命令，然後再次執行部署指令碼。此命令只會變更目前 PowerShell 程序的執行原則。

    Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass

  9. 當指令碼執行時，輸入 1 以啟動 1. Create Key Vault 選項。

    此選項會在資源群組尚不存在時建立它，並部署已啟用 RBAC 授權的 Azure Key Vault。相較於舊版存取原則，RBAC 授權是控制保存庫秘密存取的建議模型。

  10. 輸入 2 以執行 2. Assign role 選項。這會將 Key Vault Secrets Officer 角色指派給您的帳戶，讓您可以使用 Microsoft Entra 驗證來讀取、建立、更新及刪除秘密。

  11. 輸入 3 以執行 3. Store secrets 選項。這會在保存庫中儲存兩個範例秘密：模型端點的 API 金鑰（openai-api-key）以及資料庫連線字串（cosmosdb-connection-string）。兩者都會加上用於環境與服務識別的中繼資料標籤。

  12. 輸入 4 以執行 4. Check deployment status 選項。確認保存庫狀態顯示為 Succeeded、角色已指派，且秘密已儲存後再繼續。如果保存庫仍在佈建中，請稍候再試。

  13. 輸入 5 以執行 5. Retrieve connection info 選項。這會建立應用程式所需、包含 Key Vault URL 的環境變數檔案。

  14. 輸入 6 以結束部署指令碼。

  15. 執行適當的命令，將前一步建立的檔案中的環境變數載入到您的終端機工作階段中。

    Bash

    source .env

    PowerShell

    . .\.env.ps1

    Note: 請保持終端機開啟。如果您將其關閉並建立新的終端機，則需要再次執行此命令以重新載入環境變數。

加入擷取秘密的程式碼

在本節中，您將加入程式碼，從保存庫擷取兩個秘密並傳回其中繼資料。此函式示範如何存取秘密值、版本識別碼、內容類型、建立日期與自訂標籤。

此函式會針對每個秘密名稱呼叫 get_secret()，回傳秘密值以及包含中繼資料的 properties 物件。它會針對遺失的秘密處理 ResourceNotFoundError，並針對授權或網路問題處理 HttpResponseError。截斷後的值可避免完整認證出現在 UI 中，同時仍可確認秘密已成功擷取。

  1. 找出 # BEGIN RETRIEVE SECRETS FUNCTION 註解，並在該註解下方加入下列程式碼。請務必確認程式碼對齊正確。

    def retrieve_secrets():
    """Retrieve secrets and display their metadata."""
    client = get_client()
    results = []

    secret_names = ["openai-api-key", "cosmosdb-connection-string"]

    for name in secret_names:
        try:
            # get_secret returns the secret value and its properties,
            # including version, content type, creation date, and tags
            secret = client.get_secret(name)
            results.append({
                "name": secret.name,
                "value": secret.value[:20] + "..." if len(secret.value) > 20 else secret.value,
                "version": secret.properties.version,
                "content_type": secret.properties.content_type,
                "created_on": str(secret.properties.created_on),
                "tags": secret.properties.tags or {},
                "status": "retrieved"
            })
        except ResourceNotFoundError:
            results.append({
                "name": name,
                "value": None,
                "version": None,
                "content_type": None,
                "created_on": None,
                "tags": {},
                "status": "not found"
            })
        except HttpResponseError as e:
            results.append({
                "name": name,
                "value": None,
                "version": None,
                "content_type": None,
                "created_on": None,
                "tags": {},
                "status": f"error: {e.message}"
            })

    return results

  2. 花幾分鐘檢視這段程式碼。

加入列出秘密屬性的程式碼

在本節中，您將加入程式碼，在不擷取值的情況下列出保存庫中所有秘密的屬性。這遵循最小權限原則，只公開名稱、啟用狀態、內容類型與時間戳記等中繼資料。

此函式會呼叫 list_properties_of_secrets()，它會傳回可迭代的秘密屬性物件。不同於 get_secret()，此方法不會傳回秘密值，因此很適合用於盤點與稽核作業，讓您在不存取內容的情況下了解有哪些秘密存在。

  1. 找出 # BEGIN LIST SECRETS FUNCTION 註解，並在該註解下方加入下列程式碼。請務必確認程式碼對齊正確。

    def list_secret_properties():
    """List all secret properties without retrieving values."""
    client = get_client()
    results = []

    # list_properties_of_secrets returns metadata for every secret
    # in the vault without exposing the secret values, which follows
    # the principle of least privilege
    for prop in client.list_properties_of_secrets():
        results.append({
            "name": prop.name,
            "enabled": prop.enabled,
            "content_type": prop.content_type,
            "created_on": str(prop.created_on),
            "updated_on": str(prop.updated_on)
        })

    return results

  2. 儲存您的變更，並花幾分鐘檢視程式碼。

加入建立新秘密版本的程式碼

在本節中，您將加入程式碼來建立秘密的新版本，以模擬認證輪替。此函式會擷取目前版本、使用 set_secret() 寫入新值，然後再次擷取秘密以確認更新。

此函式使用 set_secret() 為既有秘密名稱寫入新值，會在保留前一版本的同時自動建立新版本。先前版本仍可透過其版本 ID 存取，但未提供版本參數的 get_secret() 一律會傳回最新版本。此函式也會將更新後的標籤附加到新版本上，以追蹤輪替中繼資料。

  1. 找出 # BEGIN CREATE SECRET VERSION FUNCTION 註解，並在該註解下方加入下列程式碼。請務必確認程式碼對齊正確。

    def create_secret_version(secret_name, new_value):
    """Create a new version of a secret and verify the update."""
    client = get_client()

    # Retrieve the current version before updating
    try:
        current = client.get_secret(secret_name)
        old_version = current.properties.version
        old_value = current.value[:20] + "..." if len(current.value) > 20 else current.value
    except ResourceNotFoundError:
        old_version = None
        old_value = None

    # set_secret creates a new version of the secret. The previous
    # version is preserved and can still be retrieved by version ID.
    client.set_secret(
        secret_name,
        new_value,
        content_type="text/plain",
        tags={"environment": "development", "rotated": "true"}
    )

    # Confirm the update by retrieving the secret again —
    # get_secret always returns the latest version
    confirmed = client.get_secret(secret_name)

    return {
        "name": secret_name,
        "old_version": old_version,
        "old_value": old_value,
        "new_version": confirmed.properties.version,
        "new_value": confirmed.value[:20] + "..." if len(confirmed.value) > 20 else confirmed.value,
        "created_on": str(confirmed.properties.created_on),
        "tags": confirmed.properties.tags or {}
    }

  2. 儲存您的變更，並花幾分鐘檢視程式碼。

加入快取式秘密擷取的程式碼

在本節中，您將加入實作以時間為基礎快取的程式碼，以降低秘密被頻繁存取時的 Key Vault API 呼叫次數。此快取會以可設定的存留時間 (TTL) 將秘密值儲存在記憶體中，並追蹤快取命中與未命中。

此函式使用 time.monotonic() 追蹤經過時間，建立一個 TTL 為 30 秒、以字典為基礎的快取。它會模擬五輪存取兩個秘密。第一輪會產生快取未命中，因為快取一開始是空的，沒有可回傳的項目，因此程式碼會從 Key Vault 擷取每個秘密並加以儲存。TTL 期間內後續的輪次會找到快取項目並直接回傳，而不需要進行 API 呼叫。存取記錄會顯示每次命中或未命中，摘要則會比較 API 呼叫總數與存取總數，以展示效率提升。

  1. 找出 # BEGIN CACHED RETRIEVAL FUNCTION 註解，並在該註解下方加入下列程式碼。請務必確認程式碼對齊正確。

    def cached_retrieval():
    """Demonstrate time-based caching to reduce Key Vault API calls."""
    client = get_client()
    cache = {}
    cache_ttl = 30
    vault_calls = 0
    access_log = []

    secret_names = ["openai-api-key", "cosmosdb-connection-string"]

    # Simulate five rounds of secret access. The first round fetches
    # from Key Vault (cache miss), and subsequent rounds return the
    # cached value if the TTL has not expired.
    for i in range(5):
        for name in secret_names:
            cached = cache.get(name)
            now = time.monotonic()

            if cached and (now - cached["timestamp"]) < cache_ttl:
                access_log.append({
                    "round": i + 1,
                    "secret": name,
                    "result": "cache hit",
                    "value": cached["value"]
                })
            else:
                secret = client.get_secret(name)
                vault_calls += 1
                truncated = secret.value[:20] + "..." if len(secret.value) > 20 else secret.value
                cache[name] = {
                    "value": truncated,
                    "timestamp": now
                }
                access_log.append({
                    "round": i + 1,
                    "secret": name,
                    "result": "cache miss",
                    "value": truncated
                })

    return {
        "access_log": access_log,
        "vault_calls": vault_calls,
        "total_accesses": len(access_log),
        "cache_ttl_seconds": cache_ttl
    }

  2. 儲存您的變更，並花幾分鐘檢視程式碼。

執行應用程式

在本節中，您將執行完成的 Flask 應用程式，以進行各種 Key Vault 秘密管理作業。此應用程式提供 Web 介面，讓您可以擷取秘密、列出其屬性、建立新版本，以及測試快取式擷取。

  1. 在終端機中執行下列命令以啟動應用程式。如有需要，請先參考本練習前面的命令來啟用環境，再執行此命令。如果您已離開 client 目錄，請先執行 cd client。

    python app.py

  2. 開啟瀏覽器並前往 http://localhost:5000 以存取應用程式。

  3. 在左側面板中選取 Retrieve Secrets。這會擷取儲存在保存庫中的兩個秘密，並在右側面板顯示其中繼資料，包括秘密名稱、截斷後的值、版本識別碼、內容類型、建立日期以及任何自訂標籤。兩個秘密都應顯示狀態為 retrieved。

  4. 選取 List Secret Properties。這會在不暴露值的情況下列出保存庫中所有秘密的屬性。結果會顯示每個秘密的名稱、啟用狀態、內容類型、建立日期與上次更新日期。此作業適用於盤點與稽核情境。

  5. 選取 Create New Version。這會為 openai-api-key 秘密建立一個具有隨機產生值的新版本，以模擬認證輪替。結果會同時顯示先前的版本和值，以及新的版本和值，確認 set_secret() 會在保留舊版本的同時建立新版本。

  6. 在左側面板中選取 Retrieve Secrets，以確認秘密已更新。

  7. 選取 Run Cached Retrieval。這會使用 30 秒 TTL 快取，模擬五輪存取兩個秘密。第一輪會顯示兩次快取未命中（每個秘密一次），因為值需要從 Key Vault 擷取。其餘輪次會顯示快取命中，因為 TTL 尚未過期。摘要會確認在總共 10 次存取中，僅進行了 2 次 Key Vault API 呼叫。

疑難排解

如果您在完成本練習時遇到問題，請嘗試以下疑難排解步驟：

驗證 Azure Key Vault 部署

  • 前往 Azure portal 並找到您的資源群組。
  • 確認 Key Vault 的 Provisioning State 顯示為 Succeeded。
  • 確認保存庫已啟用 RBAC 授權（而非 access policy 模式）。

檢查秘密

  • 執行部署指令碼中的 Check deployment status 選項，以確認秘密已成功儲存。
  • 如果缺少秘密，請再次執行 Store secrets 選項。

檢查程式碼完整性與縮排

  • 請確認所有程式碼區塊都已加入 keyvault_functions.py 中適當的區段，且位於正確的 BEGIN/END 註解標記之間。
  • 確認 Python 縮排一致（使用空格，不要使用 Tab），且所有程式碼在函式內都正確對齊。
  • 確認沒有在指定區段之外誤刪或修改任何程式碼。

驗證環境變數

  • 檢查專案根目錄中是否存在 .env 檔案，且其中包含 KEY_VAULT_URL 值。
  • 確認您已執行 source .env（Bash）或 . ..env.ps1（PowerShell），將環境變數載入到終端機工作階段中。
  • 如果變數是空的，請重新執行 source .env（Bash）或 . ..env.ps1（PowerShell）。

檢查驗證

  • 執行 az account show，確認您已登入 Azure CLI。
  • 請透過檢查 Azure portal 中的角色指派，或再次執行部署指令碼中的角色指派選項，確認 Key Vault Secrets Officer 角色已指派給您的帳戶。

檢查 Python 環境與相依性

  • 在執行應用程式前，請確認虛擬環境已啟用。
  • 執行 pip list，確認 requirements.txt 中的所有套件都已成功安裝。
