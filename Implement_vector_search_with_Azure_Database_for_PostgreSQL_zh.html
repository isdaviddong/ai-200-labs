在 Azure Database for PostgreSQL 上實作向量搜尋

在本練習中，您將使用 Azure Database for PostgreSQL 與 pgvector 擴充功能建置產品相似度搜尋應用程式。您會啟用向量儲存功能、建立含有嵌入向量的產品資料庫結構描述、透過 Flask Web 應用程式載入範例資料，並執行相似度搜尋以找出相關產品。此模式可作為建議系統、語意搜尋功能及其他 AI 驅動應用程式的基礎。

本練習將執行的工作：

  • 下載專案起始檔案並設定部署指令碼
  • 部署使用 Microsoft Entra 驗證的 Azure Database for PostgreSQL Flexible Server
  • 在伺服器部署期間完成 Flask 應用程式程式碼
  • 啟用 pgvector 擴充功能並建立 products 資料表結構描述
  • 執行 Flask 應用程式以載入產品並進行相似度搜尋
  • 新增產品並觀察相似度結果如何變化

本練習約需 30 分鐘完成。

開始之前

若要完成本練習，您需要：

  • 具備部署所需 Azure 服務權限的 Azure 訂用帳戶。如果您尚未擁有，您可以 註冊一個。
  • Visual Studio Code 安裝於其中一個 支援的平台。
  • 最新版的 Azure CLI。
  • Python 3.12 或更新版本。
  • PostgreSQL command-line tools (psql)

下載專案起始檔案並部署 Azure 服務

在本節中，您將下載專案起始檔案，並使用指令碼將所需服務部署到您的 Azure 訂用帳戶。PostgreSQL 伺服器部署需要幾分鐘才能完成。

  1. 開啟瀏覽器並輸入下列 URL 以下載起始檔案。該檔案會儲存在您的預設下載位置。

    https://github.com/MicrosoftLearning/mslearn-azure-ai/raw/main/downloads/python/postgresql-vector-search-python.zip

  2. 將該檔案複製或移動到系統中您想要進行專案作業的位置，然後將它解壓縮到資料夾中。

  3. 啟動 Visual Studio Code (VS Code)，並在選單中選取 File > Open Folder... ，然後選擇包含專案檔案的資料夾。

  4. 此專案同時包含 Bash（azdeploy.sh）與 PowerShell（azdeploy.ps1）的部署指令碼。開啟適合您環境的檔案，並變更指令碼頂端的兩個值以符合您的需求，然後儲存變更。 Note: 請勿變更指令碼中的其他任何內容。

    "<your-resource-group-name>" # Resource Group name
"<your-azure-region>" # Azure region for the resources

  5. 在功能表列中選取 Terminal > New Terminal ，以在 VS Code 中開啟終端機視窗。

  6. 執行下列命令登入您的 Azure 帳戶。依照提示選取要用於本練習的 Azure 帳戶與訂用帳戶。

    az login

  7. 執行下列命令，確認您的訂用帳戶具有本練習所需的資源提供者。

    az provider register --namespace Microsoft.DBforPostgreSQL

在本節中，您將執行部署指令碼以部署 PostgreSQL 伺服器並設定驗證。

  1. 請確認您位於專案的根目錄，並在終端機中執行適當的命令以啟動部署指令碼。

    Bash

    bash azdeploy.sh

    PowerShell

    ./azdeploy.ps1

    Note: 如果 PowerShell 因指令碼未經數位簽章而封鎖它，請在同一個終端機工作階段中執行下列命令，然後再次執行部署指令碼。此命令只會變更目前 PowerShell 處理程序的執行原則。

    Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass

  2. 當指令碼選單出現時，輸入 1 以啟動 Create PostgreSQL server with Entra authentication 選項。這會建立已啟用僅限 Entra 驗證的伺服器。 Note: 部署可能需要 5 到 10 分鐘才能完成。

    IMPORTANT: 在整個練習期間，請讓執行部署的終端機保持開啟。當部署在終端機中繼續進行時，您可以先進行下一節。

完成用戶端應用程式

在本節中，您將完成 app.py 檔案，方法是新增會與 PostgreSQL 資料庫互動的路由處理常式。這些路由可處理載入範例產品、執行相似度搜尋，以及新增產品。Flask 應用程式提供 Web 介面來測試向量相似度搜尋。

  1. 在 VS Code 中開啟 client/app.py 檔案。

  2. 搜尋 BEGIN LOAD DATA SECTION 註解，並在該註解後方直接加入下列程式碼。此路由會從 JSON 檔案載入產品，並連同其嵌入向量插入資料庫。

    @app.route("/load-data", methods=["POST"])
def load_data():
    """Load sample products into the database."""
    try:
        products = load_json_file("sample_products.json")

        with get_connection() as conn:
            with conn.cursor() as cur:
                for product in products:
                    # Check if product already exists
                    cur.execute("SELECT id FROM products WHERE name = %s", (product["name"],))
                    if cur.fetchone():
                        continue

                    # Format embedding as PostgreSQL array (pgvector expects bracket notation)
                    embedding_str = "[" + ",".join(str(x) for x in product["embedding"]) + "]"

                    cur.execute("""
                        INSERT INTO products (name, category, description, price, embedding)
                        VALUES (%s, %s, %s, %s, %s)
                    """, (
                        product["name"],
                        product["category"],
                        product["description"],
                        product["price"],
                        embedding_str
                    ))
                # Commit all inserts in a single transaction
                conn.commit()

        flash(f"Successfully loaded {len(products)} sample products!", "success")
    except Exception as e:
        flash(f"Error loading data: {str(e)}", "error")

    return redirect(url_for("index"))

  3. 搜尋 BEGIN SEARCH SECTION 註解，並在該註解後方直接加入下列程式碼。此路由會擷取所選產品的嵌入向量，並使用餘弦距離找出相似產品。

    @app.route("/search", methods=["POST"])
def search():
    """Find products similar to the selected product using vector similarity."""
    product_id = request.form.get("product_id")

    if not product_id:
        flash("Please select a product", "error")
        return redirect(url_for("index"))

    try:
        with get_connection() as conn:
            with conn.cursor() as cur:
                # Get the embedding for the selected product
                cur.execute("SELECT embedding FROM products WHERE id = %s", (product_id,))
                row = cur.fetchone()

                if not row:
                    flash("Product not found", "error")
                    return redirect(url_for("index"))

                # Find similar products using cosine distance
                # The <=> operator is pgvector's cosine distance operator
                # Lower distance = more similar (0 = identical, 2 = opposite)
                cur.execute("""
                    SELECT id, name, category, description, price, embedding <=> %s AS distance
                    FROM products
                    WHERE id != %s
                    ORDER BY distance
                    LIMIT 5
                """, (row[0], product_id))

                results = [
                    {"id": r[0], "name": r[1], "category": r[2], "description": r[3], "price": r[4], "distance": r[5]}
                    for r in cur.fetchall()
                ]

        products = get_products()
        new_products = get_new_products()
        return render_template("index.html", products=products, new_products=new_products, results=results)

    except Exception as e:
        flash(f"Error searching: {str(e)}", "error")
        return redirect(url_for("index"))

  4. 搜尋 BEGIN ADD PRODUCT SECTION 註解，並在該註解後方直接加入下列程式碼。此路由會將 new_products.json 檔案中的產品新增到資料庫。

    @app.route("/add-product", methods=["POST"])
def add_product():
    """Add a new product from the new_products.json file."""
    product_index = request.form.get("product_index")

    if product_index is None or product_index == "":
        flash("Please select a product to add", "error")
        return redirect(url_for("index"))

    try:
        new_products = load_json_file("new_products.json")
        product = new_products[int(product_index)]

        with get_connection() as conn:
            with conn.cursor() as cur:
                # Check if product already exists
                cur.execute("SELECT id FROM products WHERE name = %s", (product["name"],))
                if cur.fetchone():
                    flash(f"Product '{product['name']}' already exists", "error")
                    return redirect(url_for("index"))

                # Format embedding as PostgreSQL array
                embedding_str = "[" + ",".join(str(x) for x in product["embedding"]) + "]"

                cur.execute("""
                    INSERT INTO products (name, category, description, price, embedding)
                    VALUES (%s, %s, %s, %s, %s)
                """, (
                    product["name"],
                    product["category"],
                    product["description"],
                    product["price"],
                    embedding_str
                ))
                conn.commit()

        flash(f"Successfully added '{product['name']}'!", "success")
    except Exception as e:
        flash(f"Error adding product: {str(e)}", "error")

    return redirect(url_for("index"))

  5. 將您的變更儲存到 app.py 檔案。

  6. 花幾分鐘檢閱應用程式中的所有程式碼。請注意每個路由如何使用 get_connection() 函式以 Microsoft Entra 驗證連線到 PostgreSQL，以及 <=> 運算子如何執行相似度搜尋的餘弦距離計算。

完成 Azure 資源部署並建立結構描述

在本節中，您將啟用 pgvector 擴充功能，並建立含有向量欄位以儲存嵌入向量的 products 資料表。此結構描述包含產品詳細資料欄位，以及用於相似度搜尋的 384 維嵌入向量。

  1. 等待 PostgreSQL 伺服器在終端機中顯示部署已完成。

  2. 在部署指令碼選單中，輸入 2 以設定 Microsoft Entra 驗證。這會將您的 Azure 帳戶設為資料庫管理員。

  3. 當前一項作業完成時，輸入 3 以啟動 Check deployment status 選項。這會確認伺服器已就緒。

  4. 輸入 4 以啟動 Retrieve connection info and access token 選項。這會建立一個包含必要環境變數的檔案。

  5. 輸入 5 以結束部署指令碼。

  6. 執行下列命令，從先前步驟建立的檔案中將環境變數載入目前的終端機工作階段。

    Bash

    source .env

    PowerShell

    . .\.env.ps1

  7. 執行下列命令以使用 psql 連線到 PostgreSQL 伺服器。此命令會使用您在前一個步驟中載入的環境變數。

    Bash

    psql "host=$DB_HOST dbname=$DB_NAME user=$DB_USER sslmode=require"

    PowerShell

    psql "host=$env:DB_HOST port=5432 dbname=$env:DB_NAME user=$env:DB_USER sslmode=require"

  8. 啟用 pgvector 擴充功能。您必須先啟用此擴充功能，才能使用向量資料型別。

    CREATE EXTENSION IF NOT EXISTS vector;

  9. 建立含有嵌入向量欄位的 products 資料表。embeddings 欄位使用 384 個維度，這與常見的 sentence transformer 模型相符。

    CREATE TABLE products (
    id SERIAL PRIMARY KEY,
    name TEXT NOT NULL,
    category TEXT,
    description TEXT,
    price NUMERIC(10, 2),
    embedding vector(330)
);

  10. 建立 HNSW 索引以支援快速相似度搜尋。此索引類型已針對向量資料上的近似最近鄰查詢進行最佳化。

    CREATE INDEX products_embedding_idx
ON products USING hnsw (embedding vector_cosine_ops)
WITH (m = 16, ef_construction = 64);

  11. 列出資料表結構以確認資料表與索引已建立。

    \d products

    您應該會看到包含 id、name、category、description、price 與 embedding 欄位的資料表結構，以及 HNSW 索引。

  12. 輸入 quit 以結束工作階段。

設定並執行 Flask 應用程式

在本節中，您將安裝 Python 相依性並執行 Flask Web 應用程式。此應用程式提供瀏覽器介面，用於載入產品、搜尋相似項目，以及將新產品新增到資料庫。

  1. 執行下列命令以切換到 client 資料夾。

    cd client

  2. 執行下列命令以建立 Python 虛擬環境。視您的環境而定，命令可能是 python 或 python3。

    python -m venv .venv

  3. 執行下列命令以啟用 Python 環境。 Note: 在 Linux/macOS 上，請使用 Bash 命令。在 Windows 上，請使用 PowerShell 命令。如果是在 Windows 上使用 Git Bash，請使用 source .venv/Scripts/activate。

    Bash

    source .venv/bin/activate

    PowerShell

    .\.venv\Scripts\Activate.ps1

  4. 執行下列命令以安裝所需的 Python 套件。 requirements.txt 檔案包含 flask 作為 Web 架構、 psycopg 用於 PostgreSQL 連線，以及 azure-identity 用於驗證。

    pip install -r requirements.txt

  5. 執行 Flask 應用程式。應用程式會在連接埠 5000 啟動，並可從瀏覽器存取。

    python app.py

    您應該會看到指出 Flask 伺服器正在執行的輸出：

    * Running on all addresses (0.0.0.0)
* Running on http://127.0.0.1:5000

  6. 開啟 Web 瀏覽器並瀏覽至 http://127.0.0.1:5000。您應該會看到產品清單為空的 Vector Search Demo 頁面。

載入產品並執行相似度搜尋

在本節中，您將使用 Web 應用程式把範例產品載入資料庫並執行相似度搜尋。這些產品包含預先計算的嵌入向量，可表示每項產品的語意意義，使應用程式能根據描述找出相似項目。

  1. 在網頁上，選取 Load Sample Products。這會將 10 個產品及其嵌入向量插入資料庫。

    您應該會看到成功訊息，而產品清單現在會顯示如「Wireless Bluetooth Headphones」、「Gaming Laptop」與「Running Shoes」等項目。

  2. 在 Find Similar Products 區段中，從下拉式清單選取 Wireless Bluetooth Headphones，然後選取 Find Similar。

    應用程式會使用向量相似度查詢資料庫，並依照與所選項目在語意上的接近程度傳回產品。由於「Noise Cancelling Earbuds」在語意上相似（兩者都是音訊裝置），因此它應該會出現在結果前段。

  3. 請嘗試選擇不同的產品，並觀察相似產品如何根據所選項目的類別與描述而改變。

新增產品並觀察變化

在本節中，您將新產品新增到資料庫，並觀察它們如何出現在相似度搜尋結果中。這示範了向量搜尋如何隨著資料變化而調整。

  1. 回到顯示 Flask 應用程式的 Web 瀏覽器。

  2. 在 Add New Product 區段中，從下拉式清單選取 Espresso Machine，然後選取 Add Product。

  3. 現在請搜尋與 Coffee Maker 相似的產品，方法是在 Find Similar Products 下拉式清單中選取它，然後選取 Find Similar。

    請注意，「Espresso Machine」現在會以較低的距離分數出現在結果中，因為這兩項產品在語意上相關（家用咖啡器具）。

  4. 從下拉式清單中新增其餘產品（Wireless Gaming Mouse 與 Fitness Tracker Band），並觀察它們如何出現在相關產品的相似度搜尋中。

總結

在本練習中，您已使用 Azure Database for PostgreSQL 與 pgvector 擴充功能建置產品相似度搜尋應用程式。您部署了使用 Microsoft Entra 驗證的 PostgreSQL Flexible Server、啟用了 pgvector 擴充功能，並建立了一個包含 384 維向量欄位以儲存嵌入向量的 products 資料表。您新增了 HNSW 索引來最佳化相似度查詢，接著使用 Flask Web 應用程式載入範例產品，並利用餘弦距離運算子（<=>）執行向量相似度搜尋。此模式示範如何建置建議系統與語意搜尋功能，以根據語意意義而非精確關鍵字比對來尋找相關項目。

清除資源

既然您已完成本練習，應刪除所建立的雲端資源，以避免不必要的資源使用。

  1. 在 VS Code 終端機中執行下列命令以刪除資源群組及群組中的所有資源。請將 <rg-name> 替換為您稍早在練習中選擇的名稱。此命令會在 Azure 中啟動背景工作以刪除資源群組。

    az group delete --name <rg-name> --no-wait --yes

CAUTION: 刪除資源群組會刪除其中包含的所有資源。如果您為本練習選用了現有的資源群組，則此練習範圍外的任何既有資源也會一併刪除。

疑難排解

如果您在本練習期間遇到問題，請嘗試下列步驟：

psql 連線失敗

  • 確認 .env 檔案已透過執行部署指令碼選項 4 建立
  • 確認您已執行 source .env （Bash）或 . ..env.ps1 （PowerShell）以載入環境變數
  • 存取權杖約在一小時後過期；請再次執行部署指令碼選項 4 以產生新的權杖
  • 執行部署指令碼選項 3 來確認伺服器已就緒

存取遭拒或驗證錯誤

  • 確認已透過執行部署指令碼選項 2 設定 Microsoft Entra 系統管理員
  • 確認 PGPASSWORD 已在您的終端機工作階段中正確設定
  • 確認您使用正確的 DB_USER 值（您的 Azure 帳戶電子郵件）

Flask 應用程式無法啟動

  • 確認 Python 虛擬環境已啟用（您應該會在終端機提示字元中看到 (.venv) ）
  • 確認已安裝相依性： pip install -r requirements.txt
  • 確認三個路由函式都已正確加入 app.py

Flask 中的資料庫連線錯誤

  • 確認執行 Flask 的終端機已載入環境變數
  • 確認可透過 psql 連線並執行 \d products 來驗證 products 資料表存在
  • 確認已啟用 pgvector 擴充功能： CREATE EXTENSION IF NOT EXISTS vector;

載入後未顯示任何產品

  • 檢查 Flask 終端機中的錯誤訊息
  • 確認 sample_products.json 檔案存在於 client 資料夾中
  • 確認已使用正確的結構描述建立 products 資料表

Python venv 啟用問題

  • 在 Linux/macOS 上，請使用： source .venv/bin/activate
  • 在 Windows PowerShell 上，請使用： ..venv\Scripts\Activate.ps1
  • 如果在 Windows 上使用 Git Bash，請使用： source .venv/Scripts/activate
