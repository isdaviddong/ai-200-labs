使用 ACR Tasks 建置並執行容器映像

在本練習中，您會使用 Azure Container Registry (ACR) Tasks 完全在雲端建置及管理容器映像，而不需要在本機安裝 Docker。

本練習中將完成下列工作：

  • 下載專案起始檔
  • 部署 Azure Container Registry
  • 使用 ACR Tasks 建置並驗證容器映像
  • 管理映像版本並保護正式環境映像

本練習約需 30 分鐘完成。

重要： Azure Container Registry task runs 目前不支援 Azure 免費額度。本練習需要 Pay-As-You-Go 或其他付費方案。

開始之前

若要完成本練習，您需要：

  • 具備部署所需 Azure 服務權限的 Azure 訂閱。如果您還沒有，可以 註冊一個。
  • Visual Studio Code ，可在下列其中一個 支援的平台上執行。
  • 最新版的 Azure CLI。
  • 選用： Python 3.12 或更新版本。

下載專案起始檔並部署 Azure 服務

在本節中，您會下載專案起始檔，並使用指令碼將所需服務部署到 Azure 訂閱。Azure Container Registry 的部署需要幾分鐘才能完成。

  1. 開啟瀏覽器並輸入下列 URL 以下載起始檔。檔案會儲存在您的預設下載位置。

    https://github.com/MicrosoftLearning/mslearn-azure-ai/raw/main/downloads/python/acr-tasks-python.zip

  2. 將檔案複製或移動到您希望進行專案作業的系統位置，然後將其解壓縮到資料夾中。

  3. 啟動 Visual Studio Code (VS Code)，並選取 File > Open Folder... ，然後選擇包含專案檔案的資料夾。

  4. 此專案同時包含 Bash（azdeploy.sh）與 PowerShell（azdeploy.ps1）。開啟符合您環境的檔案，將指令碼頂端的兩個值修改為符合您的需求，然後儲存變更。 注意： 不要變更指令碼中的其他任何內容。

    "<your-resource-group-name>" # Resource Group name
"<your-azure-region>" # Azure region for the resources

  5. 在功能表列中選取 Terminal > New Terminal ，在 VS Code 中開啟終端機視窗。

  6. 執行下列命令登入您的 Azure 帳戶。依照提示選取本練習要使用的 Azure 帳戶與訂閱。

    az login

  7. 執行下列命令，以確保您的訂閱具有安裝 Azure Container Registry (ACR) 所需的資源提供者。

    az provider register --namespace Microsoft.ContainerRegistry

  8. 請確認您位於專案根目錄，並在終端機中執行對應命令來啟動部署指令碼。部署指令碼會部署 ACR，並建立本練習所需環境變數的檔案。

    Bash

    bash azdeploy.sh

    PowerShell

    ./azdeploy.ps1

    注意： 如果 PowerShell 因指令碼未經數位簽署而封鎖它，請在同一個終端機工作階段中執行下列命令，然後再次執行部署指令碼。此命令只會變更目前 PowerShell 處理序的執行原則。

    Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass

  9. 執行對應命令，將環境變數載入目前的終端機工作階段。

    Bash

    source .env

    PowerShell

    . .\.env.ps1

    注意： 請保持終端機開啟。如果您將它關閉並建立新的終端機，可能需要再次執行命令來建立環境變數。

使用 ACR Tasks 建置映像

在本節中，您會使用快速工作在 Azure 中建置映像，而不需要本機安裝 Docker。 az acr build 命令會上傳您的原始檔，在雲端建置映像，並將其推送到您的登錄。

  1. 執行下列命令來建置映像並推送到您的登錄。整個建置程序都在 Azure 中完成，不需要本機安裝 Docker。

    Bash

    az acr build \
    --registry $ACR_NAME \
    --image inference-api:v1.0.0 \
    ./api

    PowerShell

    az acr build `
    --registry $env:ACR_NAME `
    --image inference-api:v1.0.0 `
    ./api

  2. 觀察 ACR Tasks 的輸出：

    • 封裝並將您的來源內容上傳到 Azure
    • 將建置工作排入佇列並啟動
    • 串流顯示 Docker 建置輸出中的每一層
    • 將完成的映像推送到您的登錄
    • 回報映像摘要與工作狀態

驗證登錄中的映像

在本節中，您會透過列出存放庫與標籤來確認映像存在於您的登錄中。

  1. 執行下列命令以列出登錄中的所有存放庫。

    Bash

    az acr repository list --name $ACR_NAME --output table

    PowerShell

    az acr repository list --name $env:ACR_NAME --output table

    輸出會顯示 inference-api 存放庫。

  2. 執行下列命令以列出 inference-api 存放庫的標籤。

    Bash

    az acr repository show-tags \
    --name $ACR_NAME \
    --repository inference-api \
    --output table

    PowerShell

    az acr repository show-tags `
    --name $env:ACR_NAME `
    --repository inference-api `
    --output table

    輸出會顯示 v1.0.0 這個標籤是您在建置期間指定的。

  3. 執行下列命令以檢視詳細的資訊清單資訊，包括摘要。

    Bash

    az acr manifest list-metadata \
    --registry $ACR_NAME \
    --name inference-api \
    --output table

    PowerShell

    az acr manifest list-metadata `
    --registry $env:ACR_NAME `
    --name inference-api `
    --output table

    請記下摘要值。無論標籤為何，此 SHA-256 雜湊值都能唯一識別您的映像。

使用 ACR Tasks 執行映像

在本節中，您會使用 az acr run 命令在已建置的映像內執行命令，並驗證其可正常運作。

  1. 執行下列命令以驗證 Flask 應用程式可在容器中正確載入。

    Bash

    az acr run \
    --registry $ACR_NAME \
    --cmd "$ACR_NAME.azurecr.io/inference-api:v1.0.0 python -c 'from app import app'" \
    /dev/null

    PowerShell

    az acr run `
    --registry $env:ACR_NAME `
    --cmd "$env:ACR_NAME.azurecr.io/inference-api:v1.0.0 python -c 'from app import app'" `
    /dev/null

    輸出包含下載映像時的 Docker pull 進度。成功執行時會以 Run ID: xxx was successful after xxx。這表示容器可正確執行，且 Flask 應用程式可在沒有錯誤的情況下匯入。

使用不同標籤建置

在本節中，您會使用不同標籤建置映像的新版本，以了解登錄如何維護多個版本。

  1. 執行下列命令，使用新的版本標籤再次建置映像。

    Bash

    az acr build \
    --registry $ACR_NAME \
    --image inference-api:v1.1.0 \
    ./api

    PowerShell

    az acr build `
    --registry $env:ACR_NAME `
    --image inference-api:v1.1.0 `
    ./api

  2. 執行下列命令以列出所有標籤並查看兩個版本。

    Bash

    az acr repository show-tags \
    --name $ACR_NAME \
    --repository inference-api \
    --output table

    PowerShell

    az acr repository show-tags `
    --name $env:ACR_NAME `
    --repository inference-api `
    --output table

    在輸出中， v1.0.0 以及 v1.1.0 都會出現，顯示登錄如何維護多個版本。

檢視建置歷程並鎖定正式環境映像

在本節中，您會檢閱 ACR 工作執行歷程，並鎖定映像以防止意外變更。

  1. 執行下列命令以檢閱 ACR 工作執行歷程，查看您已執行的所有建置。

    Bash

    az acr task list-runs \
    --registry $ACR_NAME \
    --output table

    PowerShell

    az acr task list-runs `
    --registry $env:ACR_NAME `
    --output table

    輸出會顯示每個建置工作的執行 ID、狀態、觸發類型與持續時間。這份歷程有助於您追蹤建置並診斷問題。

  2. 執行下列命令以鎖定您的 v1.0.0 映像，防止意外刪除或修改。

    Bash

    az acr repository update \
    --name $ACR_NAME \
    --image inference-api:v1.0.0 \
    --write-enabled false

    PowerShell

    az acr repository update `
    --name $env:ACR_NAME `
    --image inference-api:v1.0.0 `
    --write-enabled false

  3. 執行下列命令以驗證鎖定是否已生效。

    Bash

    az acr repository show \
    --name $ACR_NAME \
    --image inference-api:v1.0.0 \

    PowerShell

    az acr repository show `
    --name $env:ACR_NAME `
    --image inference-api:v1.0.0 `

    欄位 writeEnabled 顯示 False，表示映像已受到保護。

清除資源

完成本練習後，您應刪除所建立的雲端資源，以避免不必要的資源使用。

  1. 在 VS Code 終端機中執行下列命令以刪除資源群組及其中所有資源。將 <rg-name> 替換為您在本練習稍早選擇的名稱。此命令會在 Azure 中啟動背景工作來刪除資源群組。

    az group delete --name <rg-name> --no-wait --yes

注意： 刪除資源群組會刪除其中包含的所有資源。如果您在本練習中選擇的是現有資源群組，則本練習範圍外的現有資源也會一併被刪除。

疑難排解

若您在完成本練習時遇到問題，請嘗試下列疑難排解步驟：

驗證 Azure 驗證與環境變數

  • 執行 az account show 以確認您已登入正確的 Azure 訂閱。
  • 執行下列命令以確認環境變數已設定： echo $ACR_NAME （Bash）或 $env:ACR_NAME （PowerShell）。
  • 如果變數為空，請重新執行 source .env （Bash）或 . ..env.ps1 （PowerShell）。

驗證 ACR 部署

  • 前往 Azure portal 並找到您的資源群組。
  • 確認 Azure Container Registry 存在，且 Provisioning State 為 Succeeded.
  • 執行 az acr list --output table 以確認您的登錄可供存取。

排解建置失敗

  • 檢查建置輸出中的錯誤訊息；常見問題包括 Dockerfile 遺失或檔案路徑不正確。
  • 確認您是從專案根目錄執行命令（ api 資料夾所在的位置）。
  • 執行 az acr task list-runs --registry $ACR_NAME --output table 以查看最近建置的狀態。
