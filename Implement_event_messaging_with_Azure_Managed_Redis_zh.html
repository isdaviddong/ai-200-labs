在 Azure Managed Redis 中發佈與訂閱事件

在本練習中，您將建立 Azure Managed Redis 資源，並完成一個以主控台為基礎的發佈者應用程式與訂閱者應用程式的程式碼。發佈者應用程式會將事件訊息傳送到 Redis 通道，而訂閱者應用程式則會使用以 tkinter建置的圖形介面來接收這些訊息。您將實作核心的 pub/sub 模式，包括直接通道訂閱、萬用字元模式比對、訊息格式化，以及背景訊息接聽。

本練習將執行的工作：

  • 下載專案起始檔案
  • 建立 Azure Managed Redis 資源
  • 加入程式碼以完成發佈者與訂閱者應用程式
  • 執行發佈者與訂閱者應用程式以傳送與接收訊息

本練習約需 30 分鐘完成。

開始之前

若要完成本練習，您需要：

  • Azure 訂用帳戶。如果您尚未擁有，您可以 註冊一個。
  • Visual Studio Code 安裝於其中一個 支援的平台。
  • Python 3.12 或更新版本。
  • 最新版的 Azure CLI。
  • Azure CLI 的 redisenterprise 延伸模組。您可以執行 az extension add --name redisenterprise 命令來安裝它。

    def connect_to_redis() -> redis.Redis:
    """Establish connection to Azure Managed Redis using SSL encryption and authentication"""

    try:
        redis_host = os.getenv("REDIS_HOST")
        redis_key = os.getenv("REDIS_KEY")

        r = redis.Redis(
            host=redis_host,
            port=10000,
            ssl=True,
            decode_responses=True,
            password=redis_key,
            socket_timeout=30,
            socket_connect_timeout=30,
        )

        # Test connection
        r.ping()  # Verify Redis connectivity
        return r

    except redis.ConnectionError as e:
        print(f"[x] Connection error: {e}")
        print("Check if Redis host and port are correct, and ensure network connectivity")
        sys.exit(1)
    except redis.AuthenticationError as e:

        print("Make sure the access key is correct")
        sys.exit(1)
    except Exception as e:
        print(f"[x] Unexpected error: {e}")
        sys.exit(1)

加入發佈訊息的程式碼

在本節中，您將加入程式碼，使用 publish() 方法將事件訊息發佈到特定的 Redis 通道。發佈者會傳送包含訂單資訊等事件資料的 JSON 格式訊息。每次呼叫 publish() 都會回傳收到該訊息的有效訂閱者數量，讓您可以確認訊息已成功傳遞。這就是 pub/sub 模式的核心：發佈者不需要知道個別訂閱者的詳細資訊。

  1. 尋找 # BEGIN PUBLISH MESSAGE CODE SECTION 註解，並在該註解下方加入下列程式碼。請務必確認程式碼縮排正確。

    def publish_order_created(r: redis.Redis) -> None:
    """Publish an order created event using r.publish() to the 'orders:created' channel"""
    clear_screen()
    print("=" * 60)
    print("Publishing: Order Created Event")
    print("=" * 60)

    order_data = {
        "event": "order_created",
        "order_id": f"ORD-{datetime.now().strftime('%Y%m%d%H%M%S')}",
        "customer": "Jane Doe",
        "total": 129.99,
        "timestamp": datetime.now().isoformat()
    }

    message = json.dumps(order_data)
    channel = "orders:created"

    # Publish message and get subscriber count
    subscribers = r.publish(channel, message)  # Send message to channel, returns number of subscribers that received it

    print(f"\n[>] Published to channel: '{channel}'")
    print(f"[#] Active subscribers: {subscribers}")
    print(f"\n[i] Message content:")
    print(json.dumps(order_data, indent=2))

    input("\n[+] Press Enter to continue...")

    def broadcast_to_all(r: redis.Redis) -> None:
    """Broadcast a message to all channels using r.publish() in a loop for multi-channel delivery"""
    clear_screen()
    print("=" * 60)
    print("Broadcasting: System Announcement")
    print("=" * 60)

    announcement = {
        "event": "system_announcement",
        "message": "System maintenance scheduled for 2 AM",
        "priority": "high",
        "timestamp": datetime.now().isoformat()
    }

    channels = ["orders:created", "orders:shipped", "inventory:alerts", "notifications"]
    message = json.dumps(announcement)

    print(f"\n[>] Broadcasting to {len(channels)} channels...")
    print(f"Channels: {', '.join(channels)}\n")

    total_subscribers = 0
    for channel in channels:
        count = r.publish(channel, message)  # Send same message to multiple channels
        total_subscribers += count
        print(f"  - {channel}: {count} subscriber(s)")

    print(f"\n[#] Total subscribers reached: {total_subscribers}")
    print(f"\n[i] Message content:")
    print(json.dumps(announcement, indent=2))

    input("\n[+] Press Enter to continue...")

加入訊息格式化程式碼

在本節中，您將加入程式碼，格式化傳入的 pub/sub 訊息，以便在訂閱者應用程式中顯示。 format_message_gui() 函式會解析已發佈訊息中的 JSON 承載，並根據事件類型擷取相關欄位。此函式同時處理標準通道訊息與模式比對訊息，提供一致且易讀的顯示格式，方便理解透過 pub/sub 系統傳輸的資料內容。

  1. 尋找 # BEGIN MESSAGE FORMATTING CODE SECTION 註解，並在該註解下方加入下列程式碼。請務必確認程式碼縮排正確。

    def format_message_gui(message_data: dict) -> str:
    """Format message data for GUI display, parsing JSON payload and extracting relevant fields"""
    timestamp = datetime.now().strftime("%H:%M:%S")
    channel = message_data.get('channel', 'unknown')

    try:
        data = json.loads(message_data['data'])
        event_type = data.get('event', 'unknown')

        formatted = f"[{timestamp}] Message on '{channel}'\n"
        formatted += f"{'─' * 50}\n"
        formatted += f"Event: {event_type}\n"

        # Display relevant fields based on event type
        if 'order_id' in data:
            formatted += f"Order ID: {data['order_id']}\n"
        if 'customer' in data:
            formatted += f"Customer: {data['customer']}\n"
        if 'total' in data:
            formatted += f"Total: ${data['total']}\n"
        if 'tracking_number' in data:
            formatted += f"Tracking: {data['tracking_number']}\n"
        if 'product_name' in data:
            formatted += f"Product: {data['product_name']}\n"
        if 'current_stock' in data:
            formatted += f"Stock Level: {data['current_stock']}\n"
        if 'message' in data:
            formatted += f"Message: {data['message']}\n"

        formatted += f"{'─' * 50}\n"
        return formatted

    except json.JSONDecodeError:
        return f"[{timestamp}] {channel}: {message_data['data']}\n"

  2. 儲存您的變更

加入訊息接聽程式碼

在本節中，您將加入背景接聽執行緒的程式碼，用來持續監視已訂閱通道中的傳入訊息。 listen_messages() 方法會使用具阻塞性的 pubsub.listen() 迭代器在訊息發佈時接收訊息。這示範了訂閱者如何被動等待訊息，並處理不同類型的訊息（直接通道訊息與模式比對訊息）。接聽器會在背景執行緒中運作，以避免阻塞主要應用程式，同時仍能接收即時訊息更新。

  1. 尋找 # BEGIN MESSAGE LISTENER CODE SECTION 註解，並在該註解下方加入下列程式碼。請務必確認程式碼縮排正確。

    def listen_messages(self):
    """Background thread to listen for messages using pubsub.listen() blocking iterator"""
    self.listener_active = True

    try:
        for message in self.pubsub.listen():  # Listen for published messages (blocking)
            if not self.listening:
                break

            if message['type'] == 'message':
                formatted = format_message_gui(message)
                self.message_queue.put(formatted)

            elif message['type'] == 'pmessage':
                # Pattern-based subscription
                timestamp = datetime.now().strftime("%H:%M:%S")
                pattern = message['pattern']
                channel = message['channel']
                try:
                    data = json.loads(message['data'])
                    event_type = data.get('event', 'unknown')
                    msg = f"[{timestamp}] Pattern '{pattern}' matched '{channel}'\n"
                    msg += f"{'-' * 50}\n"
                    msg += f"Event: {event_type}\n"
                    msg += f"Full message: {json.dumps(data, indent=2)}\n"
                    msg += f"{'-' * 50}\n"
                    self.message_queue.put(msg)
                except json.JSONDecodeError:
                    self.message_queue.put(f"[{timestamp}] Pattern '{pattern}': {message['data']}\n")

    except Exception as e:
        if self.listening:
            self.message_queue.put(f"[x] Listener error: {e}\n")
    finally:
        self.listener_active = False

  2. 儲存您的變更。

加入訂閱通道的程式碼

在本節中，您將加入程式碼來處理通道與模式訂閱。subscribe_to_channel() 方法會使用 pubsub.subscribe() 來註冊對特定通道的興趣，而 subscribe_to_pattern() 則使用 pubsub.psubscribe() 進行萬用字元模式比對（例如 orders:*）。這些函式示範了 Redis pub/sub 的兩種主要訂閱策略：針對特定事件的直接通道訂閱，以及具有彈性的模式型訂閱。完成訂閱後，接聽器會重新啟動，開始接收新通道上的訊息。

  1. 尋找 # BEGIN SUBSCRIBE CHANNEL/PATTERN CODE SECTION 註解，並在該註解下方加入下列程式碼。請務必確認程式碼縮排正確。

    def subscribe_to_channel(self, channel: str) -> str:
    """Subscribe to a specific channel using pubsub.subscribe() for direct messaging"""
    try:
        self.pubsub.subscribe(channel)  # Subscribe to channel
        self.restart_listener()
        return f"[+] Subscribed to channel: '{channel}'"
    except Exception as e:
        return f"[x] Error subscribing: {e}"

def subscribe_to_pattern(self, pattern: str) -> str:
    """Subscribe using a pattern with pubsub.psubscribe() for wildcard channel matching (e.g., 'orders:*')"""
    try:
        self.pubsub.psubscribe(pattern)  # Subscribe to pattern (e.g., 'orders:*')
        self.restart_listener()
        return f"[+] Subscribed to pattern: '{pattern}'"
    except Exception as e:
        return f"[x] Error subscribing: {e}"

花一些時間試著訂閱不同通道並發佈訊息。以下表格列出各個訂閱者選項的詳細說明：

訂閱者選項 說明
Subscribe to Channel 訂閱單一通道。通道選項會列在對話方塊中。
Subscribe with Pattern 訂閱多個通道。例如，使用 orders:* 模式會訂閱所有 orders 通道。
Unsubscribe from Channel 取消訂閱單一通道。注意：您無法使用此選項取消模式型訂閱。
Unsubscribe All 取消訂閱所有通道，包括模式型訂閱。
View Active Subscriptions 列出所有已訂閱的通道，包括模式型訂閱。

清除資源

既然您已完成本練習，應刪除所建立的雲端資源，以避免不必要的資源使用。

  1. 在 VS Code 終端機中執行下列命令以刪除資源群組及群組中的所有資源。請將 <rg-name> 替換為您稍早在練習中選擇的名稱。此命令會在 Azure 中啟動背景工作以刪除資源群組。

    az group delete --name <rg-name> --no-wait --yes

CAUTION: 刪除資源群組會刪除其中包含的所有資源。如果您為本練習選用了現有的資源群組，則此練習範圍外的任何既有資源也會一併刪除。

疑難排解

如果您在完成本練習時遇到問題，請嘗試下列疑難排解步驟：

驗證 Azure Managed Redis 資源部署

  • 瀏覽至 Azure portal 並找出您的資源群組。
  • 確認 Azure Managed Redis 資源顯示的 Provisioning State 為 Succeeded。
  • 確認資源已啟用 Public network access ，且 Access keys authentication 設為 Enabled。

檢查程式碼完整性與縮排

  • 確認所有程式碼區塊都已加入正確區段，並位於對應的 BEGIN/END 註解標記之間。
  • 確認 Python 縮排一致（使用空格，不使用 Tab），且所有程式碼在函式內都正確對齊。
  • 確認在指定區段之外沒有任何程式碼被意外刪除或修改。

驗證環境變數

  • 確認 .env 檔案存在於專案資料夾中，且包含有效的 REDIS_HOST 與 REDIS_KEY 值。
  • 確認 .env 檔案位於專案根目錄中。

檢查 Python 環境與相依性

  • 確認在執行應用程式前已啟用虛擬環境。
  • 確認 requirements.txt 中的所有套件都已成功安裝，可執行 pip list。
