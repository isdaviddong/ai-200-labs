從 Azure App Configuration 擷取設定與秘密

AI 應用程式同時依賴非敏感設定（例如模型端點與批次大小）以及敏感認證（例如 API 金鑰）。Azure App Configuration 提供集中式儲存區來管理這些設定，支援以標籤為基礎的環境覆寫、用於秘密的 Key Vault 參考，以及以 Sentinel 為基礎的動態重新整理，讓應用程式無需重新啟動即可取得設定變更。

在本練習中，您將部署一個預先載入範例設定的 Azure App Configuration 儲存區與 Key Vault，並建置一個使用 Azure SDK 展示核心設定管理模式的 Python Flask Web 應用程式。您會使用標籤堆疊與自動 Key Vault 參考解析來載入設定、列出所有設定屬性與中繼資料，並觸發以 Sentinel 為基礎的重新整理，以動態取得變更。

本練習將執行的工作：

  • 下載專案起始檔
  • 建立含有範例設定的 Azure App Configuration 儲存區與 Key Vault
  • 在起始檔中加入程式碼以完成應用程式
  • 執行應用程式以進行設定作業

本練習大約需要 30 分鐘完成。

開始之前

若要完成本練習，您需要：

  • Azure 訂用帳戶。如果您還沒有，可以 註冊一個。
  • Visual Studio Code 安裝於其中一個 支援平台。
  • Python 3.12 或更新版本。
  • 最新版的 Azure CLI。

下載專案起始檔並部署 Azure App Configuration

在本節中，您將下載應用程式的起始檔，並使用指令碼將包含範例設定的 Azure App Configuration 儲存區與 Key Vault 部署到您的訂用帳戶。

  1. 開啟瀏覽器並輸入下列 URL 以下載起始檔。檔案會儲存在您的預設下載位置。

    https://github.com/MicrosoftLearning/mslearn-azure-ai/raw/main/downloads/python/app-config-python.zip

  2. 將檔案複製或移動到您系統中要進行專案作業的位置，然後將檔案解壓縮到資料夾中。

  3. 啟動 Visual Studio Code (VS Code)，然後在功能表中選取 File > Open Folder...，接著選擇包含專案檔案的資料夾。

  4. 此專案同時包含 Bash (azdeploy.sh) 與 PowerShell (azdeploy.ps1) 的部署指令碼。請開啟符合您環境的檔案，修改指令碼頂端的兩個值以符合您的需求，然後儲存變更。注意： 請勿變更指令碼中的其他任何內容。

    "<your-resource-group-name>" # Resource Group name
"<your-azure-region>" # Azure region for the resources

  5. 在功能表列中選取 Terminal > New Terminal，以在 VS Code 中開啟終端機視窗。

  6. 執行下列命令登入您的 Azure 帳戶。依照提示選取本練習要使用的 Azure 帳戶與訂用帳戶。

    az login

  7. 執行下列命令，確認您的訂用帳戶具有本練習所需的資源提供者。

    az provider register --namespace Microsoft.AppConfiguration
az provider register --namespace Microsoft.KeyVault

  8. 在終端機中執行適當的命令以啟動指令碼。

    Bash

    bash azdeploy.sh

    PowerShell

    ./azdeploy.ps1

    Note: 如果 PowerShell 因指令碼未經數位簽署而封鎖它，請在相同的終端機工作階段中執行下列命令，然後再次執行部署指令碼。此命令只會變更目前 PowerShell 程序的執行原則。

    Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass

  9. 當指令碼執行時，輸入 1 以啟動 1. Create App Configuration 選項。

    此選項會在資源群組尚不存在時建立它，並部署 Azure App Configuration 儲存區。App Configuration 提供集中式服務，讓您可將應用程式設定與程式碼分開管理。

  10. 輸入 2 以執行 2. Create Key Vault 選項。這會建立已啟用 RBAC 授權的 Azure Key Vault。Key Vault 會安全地儲存 App Configuration 參考的敏感值，例如 API 金鑰。

  11. 輸入 3 以執行 3. Assign roles 選項。這會將 App Configuration Data Owner 角色與 Key Vault Secrets Officer 角色指派給您的帳戶，讓您可以使用 Microsoft Entra 驗證來讀取、建立及更新設定與秘密。

  12. 輸入 4 以執行 4. Store settings 選項。這會在 App Configuration 儲存區中儲存組態設定，包括預設（未加標籤）值，以及用於特定環境設定的 Production 標籤覆寫值。它也會在 Key Vault 中儲存一個秘密，並在 App Configuration 中建立指向該秘密的 Key Vault 參考。最後，它會建立一個用於動態重新整理的 Sentinel 索引鍵。

  13. 輸入 5 以執行 5. Check deployment status 選項。確認 App Configuration 儲存區與 Key Vault 都顯示為 Succeeded、角色已指派，且設定已儲存後再繼續。如果資源仍在佈建中，請稍候再試。

  14. 輸入 6 以執行 6. Retrieve connection info 選項。這會建立應用程式所需、包含 App Configuration 端點 URL 的環境變數檔案。

  15. 輸入 7 以結束部署指令碼。

  16. 執行適當的命令，將前一步建立的檔案中的環境變數載入到您的終端機工作階段中。

    Bash

    source .env

    PowerShell

    . .\.env.ps1

    Note: 請保持終端機開啟。如果您將其關閉並建立新的終端機，則需要再次執行此命令以重新載入環境變數。

  1. 開啟 client/appconfig_functions.py 檔案，開始加入程式碼。

Note: 您加入應用程式的程式碼區塊應與該程式碼區段的註解相對應。

加入載入設定的程式碼

在本節中，您將加入程式碼，以使用標籤堆疊與自動 Key Vault 參考解析，從 App Configuration 儲存區載入所有組態設定。此函式會建立一個提供者，將未加標籤的預設值與 Production 標籤覆寫值合併，並以透明方式解析 Key Vault 參考。

The function calls load() with two SettingSelector entries: the first selects all unlabeled settings (using the null label filter \0), and the second selects all Production-labeled settings. Because the Production selector appears second, its values override the defaults for any matching keys. The AzureAppConfigurationKeyVaultOptions parameter tells the provider to resolve Key Vault references automatically using the same credential, so the application receives the actual secret value rather than a reference URI.

  1. 找出 # BEGIN LOAD SETTINGS FUNCTION 註解，並在該註解下方加入下列程式碼。請務必確認程式碼對齊正確。

    def load_settings():
    """Load all settings with label stacking and Key Vault reference resolution."""
    provider = get_provider()
    results = []

    # The provider resolves Key Vault references automatically and
    # applies label stacking: Production-labeled values override
    # unlabeled defaults for matching keys
    known_keys = [
        "OpenAI:Endpoint",
        "OpenAI:DeploymentName",
        "OpenAI:ApiKey",
        "Pipeline:BatchSize",
        "Pipeline:RetryCount",
        "Sentinel"
    ]

    for key in known_keys:
        try:
            value = provider[key]
            is_secret = key == "OpenAI:ApiKey"
            display_value = value[:10] + "..." if is_secret and len(value) > 10 else value
            results.append({
                "key": key,
                "value": display_value,
                "type": "Key Vault reference" if is_secret else "configuration",
                "status": "loaded"
            })
        except KeyError:
            results.append({
                "key": key,
                "value": None,
                "type": "unknown",
                "status": "not found"
            })

    return results

  2. 花幾分鐘檢視這段程式碼。

加入列出設定屬性的程式碼

在本節中，您將加入程式碼來列出 App Configuration 儲存區中所有設定的屬性。不同於會合併標籤並解析 Key Vault 參考的 load() 函式，此函式會顯示原始儲存檢視，其中包含每一筆個別設定項目，以及所有標籤與內容類型。

此函式會在管理用戶端上呼叫 list_configuration_settings()，傳回可迭代的設定物件，其中包含索引鍵、標籤、內容類型、上次修改時間戳記與唯讀狀態等中繼資料。這對於需要精確查看儲存內容的盤點與稽核作業很有幫助，包括分開的未加標籤項目與加上 Production 標籤的項目。

  1. 找出 # BEGIN LIST SETTINGS FUNCTION 註解，並在該註解下方加入下列程式碼。請務必確認程式碼對齊正確。

    def list_setting_properties():
    """List all setting properties from the App Configuration store."""
    client = get_client()
    results = []

    # list_configuration_settings returns every setting in the store
    # including all labels, showing the raw storage view rather than
    # the merged view that load() provides
    for setting in client.list_configuration_settings():
        results.append({
            "key": setting.key,
            "label": setting.label or "(no label)",
            "content_type": setting.content_type or "—",
            "last_modified": str(setting.last_modified) if setting.last_modified else "—",
            "read_only": setting.read_only
        })

    return results

  2. 儲存您的變更，並花幾分鐘檢視程式碼。

加入動態重新整理的程式碼

在本節中，您將加入示範以 Sentinel 為基礎動態重新整理的程式碼。此函式會更新一項設定並設定新的 Sentinel 值，然後在提供者上呼叫 refresh()，以在不重新啟動應用程式的情況下重新載入設定。

此函式會先擷取目前提供者的值，接著使用管理用戶端以新的隨機值更新 Pipeline:BatchSize，並將 Sentinel 索引鍵設為新的時間戳記值。Sentinel 會作為變更訊號：提供者會監看它，當其值改變時，呼叫 refresh() 就會觸發所有設定的重新載入。此函式會短暫等待重新整理間隔過去，然後呼叫 refresh()，並比較前後值以確認更新已傳播。

  1. 找出 # BEGIN REFRESH CONFIGURATION FUNCTION 註解，並在該註解下方加入下列程式碼。請務必確認程式碼對齊正確。

    def refresh_configuration():
    """Demonstrate sentinel-based dynamic refresh of configuration settings."""
    provider = get_provider()
    client = get_client()

    # Capture current values before the change
    tracked_keys = ["Pipeline:BatchSize"]
    before = {}
    for key in tracked_keys:
        try:
            before[key] = provider[key]
        except KeyError:
            before[key] = "—"

    # Update Pipeline:BatchSize with a new value to simulate a
    # configuration change, then increment the Sentinel key to
    # signal the provider that settings have changed
    import random
    new_batch = str(random.randint(100, 999))

    setting = ConfigurationSetting(
        key="Pipeline:BatchSize",
        value=new_batch,
        label="Production",
        content_type="text/plain"
    )
    client.set_configuration_setting(setting)

    # Update the Sentinel to signal the provider that settings have changed.
    # Using a timestamp ensures the value is always different from whatever
    # the provider has cached, even if settings were reset externally.
    new_sentinel = str(int(time.time()))

    sentinel_setting = ConfigurationSetting(
        key="Sentinel",
        value=new_sentinel
    )
    client.set_configuration_setting(sentinel_setting)

    # Wait briefly for the refresh interval to elapse, then call
    # refresh() to reload settings from the store
    time.sleep(2)
    provider.refresh()

    # Capture values after the refresh
    after = {}
    for key in tracked_keys:
        try:
            after[key] = provider[key]
        except KeyError:
            after[key] = "—"

    settings = []
    for key in tracked_keys:
        settings.append({
            "key": key,
            "before": before[key],
            "after": after[key],
            "changed": before[key] != after[key]
        })

    return {
        "settings": settings,
        "sentinel_value": new_sentinel,
        "new_batch_size": new_batch,
        "batch_size_updated": after["Pipeline:BatchSize"] == new_batch
    }

  2. 儲存您的變更，並花幾分鐘檢視程式碼。

  2. 執行下列命令以建立 Python 環境。

    python -m venv .venv

  3. 執行下列命令以啟用 Python 環境。注意： 在 Linux/macOS 上，請使用 Bash 命令；在 Windows 上，請使用 PowerShell 命令。如果您在 Windows 上使用 Git Bash，請使用 source .venv/Scripts/activate。

執行應用程式

在本節中，您將執行完成的 Flask 應用程式，以進行各種 App Configuration 管理作業。此應用程式提供 Web 介面，讓您可以載入設定、列出其屬性，並測試動態重新整理。

  1. 在終端機中執行下列命令以啟動應用程式。如有需要，請先參考本練習前面的命令來啟用環境，再執行此命令。如果您已離開 client 目錄，請先執行 cd client。

    python app.py

  2. 開啟瀏覽器並前往 http://localhost:5000 以存取應用程式。

  3. 在左側面板中選取 Load Settings。這會使用標籤堆疊與 Key Vault 參考解析載入所有組態設定。結果會顯示每個設定的索引鍵、值與類型。標示為 configuration 的設定是一般 App Configuration 值，而 Key Vault reference 則表示該值已從 Key Vault 秘密解析取得。

  4. 選取 List Setting Properties。這會列出儲存區中的每一筆個別設定項目，包含未加標籤的預設值與加上 Production 標籤的覆寫值，並分別顯示為不同資料列。結果會顯示每個設定的索引鍵、標籤、內容類型、上次修改時間戳記與唯讀狀態。請注意，Pipeline:BatchSize 會出現兩次：一次沒有標籤（值為 10），一次帶有 Production 標籤（值為 200）。Load Settings 的結果顯示 200，是因為加上 Production 標籤的覆寫值優先於未加標籤的預設值。

  5. 選取 Refresh Configuration。這會示範以 Sentinel 為基礎的動態重新整理。此函式會以新的隨機值更新 Pipeline:BatchSize、將 Sentinel 索引鍵設為新的時間戳記、短暫等待，然後在提供者上呼叫 refresh()。結果會顯示所追蹤設定的變更前後值，確認提供者在不重新啟動應用程式的情況下已取得變更。

疑難排解

如果您在完成本練習時遇到問題，請嘗試以下疑難排解步驟：

驗證 Azure App Configuration 部署

  • 前往 Azure portal 並找到您的資源群組。
  • 確認 App Configuration 儲存區的 Provisioning State 顯示為 Succeeded。
  • 確認 Key Vault 的 Provisioning State 顯示為 Succeeded，且已啟用 RBAC 授權。

檢查設定

  • 執行部署指令碼中的 Check deployment status 選項，以確認設定已成功儲存。
  • 如果缺少設定，請再次執行 Store settings 選項。

檢查程式碼完整性與縮排

  • 請確認所有程式碼區塊都已加入 appconfig_functions.py 中適當的區段，且位於正確的 BEGIN/END 註解標記之間。
  • 確認 Python 縮排一致（使用空格，不要使用 Tab），且所有程式碼在函式內都正確對齊。
  • 確認沒有在指定區段之外誤刪或修改任何程式碼。

驗證環境變數

  • 檢查專案根目錄中是否存在 .env 檔案，且其中包含 AZURE_APPCONFIG_ENDPOINT 值。
  • 確認您已執行 source .env（Bash）或 . ..env.ps1（PowerShell），將環境變數載入到終端機工作階段中。
  • 如果變數是空的，請重新執行 source .env（Bash）或 . ..env.ps1（PowerShell）。

檢查驗證

  • 執行 az account show，確認您已登入 Azure CLI。
  • 請透過檢查 Azure portal 中的角色指派，或再次執行部署指令碼中的角色指派選項，確認 App Configuration Data Owner 與 Key Vault Secrets Officer 角色已指派給您的帳戶。

檢查 Python 環境與相依性

  • 在執行應用程式前，請確認虛擬環境已啟用。
  • 執行 pip list，確認 requirements.txt 中的所有套件都已成功安裝。
